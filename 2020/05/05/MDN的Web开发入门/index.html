安装本地Web服务器

  • 互联网如何工作

    • 路由器、调制解调器(modem)、ISP(互联网服务提供商)、IP地址和域名、Internet和Web

      互联网借助路由器、已有的设施(电话线, 需通过调制解调器将电话信号与网络信号相互转换)、ISP(其管理特殊的路由器)以及所有的电脑构建，每台电脑对应一个IP地址。注意Internet指互联网，而Web指网络，前者为基础设施，后者为建立在前者上的服务。

  • 网页、网站、网络服务器、搜索引擎

    网页是特定类型——HTML的文档，样式信息控制观感、脚本添加交互、多媒体添加多种形式的资源；网站是共享唯一域名的网页的集合；网络服务器是用于托管网站的计算机；搜索引擎是用于搜索和访问网页的工具，其本身也是一个网站，浏览器为基础设施，用以展示网页，搜索引擎为建立在其上的服务。

  • 网络服务器

    • 硬件部分和软件部分

      硬件部分即为存储网络服务软件和网站内容(包括文档和样式)的计算机；软件部分即为控制用户对网站的访问。

    • 静态服务器和动态服务器

      静态指将托管网站的文件保持原样发送给用户；动态则通过将数据库中的内容填充到模板中来生成HTML文档，实际上服务中存储的对应文件并不为HTML格式。这样可以提高效率和降低维护成本。

    • 托管文件

      托管的好处是：一直启动和运行；一直与互联网连接；一直拥有相同的IP地址；由他人维护。

    • 通过HTTP交流(textual and stateless)

      HTTP的命令是纯文本的，而通过HTTP对文件的请求状态不会被保留，即服务器不能记住你输入的密码或者你处于请求的哪一步。

    • 静态内容和动态内容

      静态意味这保持原样的提供文件，即通过HTML、CSS和Javascript文件展现对应的网页内容；而动态则意味着从数据库提取内容，通过模板安排布局。

  • 设置本地测试服务器

    • 本地文件(异步请求和服务端代码)和远程文件

      由于本地文件有异步请求或者包含服务端语言代码，可能无法作为本地文件打开

    • 运行本地HTTP服务器（用于解决异步请求问题）

  • 在本地运行服务器端语言

    服务器端语言通常用于搭建一个动态网站，借助于一个web框架，来实现信息的高效存储和运输、定制用户体验、控制对内容的访问、存储会话和状态信息、通知和通讯、进行数据分析。以下为常用的web框架：

    • Python：Django/Flask
    • Node.js：Express
    • PHP：MAMP/AMPPS

    <p> class="editor-note">我的猫咪脾气爆:)</p>

    • | 开始标签 | 结束标签 | 内容 | 元素 |
      | :————-: | :————-: | :——-: | :——-: |

    • <html></html>根元素

一切皆盒子

html页面内大部分元素可以被看作若干层叠的盒子

  • 盒子(块)的属性：

    1. padding: 内边距，围绕着内容(如段落)的空间

    2. border: 边框，紧接着内边距的线。

    3. margin: 外边距，围绕元素外部的空间

    4. width: 元素宽度

    5. background-color: 元素内容和内边距底下的颜色

    6. color: 元素内容(文本)的颜色

    7. text-shadow: 为元素内文本设置阴影

    8. display: 设置元素的显示模式(暂略)

  • 更改页面颜色

  • 文档体格式设置

    1. width: 600px;——强制页面永远保持600px

    2. margin: 0 auto;——为margin或padding设置两个值时，第一个值代表元素的上方和下方；第二个则代表左右。auto表示水平方向上左右对称(可以理解为居中操作)。(可以根据情况设置1-4个值)

    3. backgroud-color——指定元素的背景色

    4. padding: 0 20px 20px 20px;——分别对应上0、右20、左20、下20。（补充：3个值时，第1个值对应上，2对应左和右，3对应下）

    5. border: 5px solid black——为body设置5px的黑色实现边框。

  • 定位页面主标题并添加样式

    • text-shadow属性：第1个值设置水平偏移值、2设置垂直、3设置阴影魔糊半径、4设置阴影基色

  • 图像居中

    • 通过display: block;给予内联元素img与<body>相同的块级行为，从而能够对其进行margin: 0 auto;操作使其居中

万维网是如何工作的

客户端和服务器

  • 客户端请求(requests)，服务器回应(responses)

其它方面

  • 网络连接：允许客户端发送和接收数据

  • TCP/IP：定义数据如何传输的通信协议

  • DNS：域名系统服务器类似网站通信录帮助客户端找到对应服务器获取目标网页内容

  • HTTP：定义客户端和服务器间交流的语言

  • 网页组成文件：

    1. 代码：HTML、CSS、JavaScript

    2. 资源：图像、音频、视频、文件

请求和回应过程

  1. 浏览器在域名系统(DNS)服务器上找出存放网页的服务器的实际地址

  2. 浏览器发送HTTP请求信息到服务器拷贝一份目标网页

  3. 服务器同意客户端请求，传输网页文件

  4. 浏览器加载网页

DNS解析和数据传输——详见 互联网协议
