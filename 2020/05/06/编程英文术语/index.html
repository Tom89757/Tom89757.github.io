以下内容按字母顺序排列

A

addressing mode：寻址模式，根据对操作数和/或地址的使用不同加以区分的多种寻址方式中的一种。

API：应用程序接口（Application Programming Interface），由与因特网相联的端系统提供，是软件系统不同组成部分衔接的约定。API规定了运行在一个端系统上的软件请求因特网基础设施向运行在另一个端系统上的特定目的地软件交付数据的方式。

Application Binary Interface：应用二进制及接口，用户部分的指令加上应用程序员调用的操作系统接口，定义了二进制层次可移植的计算机的标准。

ASLR：地址空间布局随机化(Address-Space Layout Randomization)。从用ASLR，每次运行时程序的不同部分，都会被加载到内存的不同区域。

B

basic block：基本块，没有分支（可能出现在末尾者除外）并且没有分支目标/分支标签（可能出现在开始者除外）的指令序列。

benchmark：基准测试程序：用于比较计算机性能的程序。

BIOS：基本输入输出系统（Basic Input/Output System），亦称为ROM BIOS、System BIOS、PC BIOS，是在通电引导阶段运行硬件初始化，以及为操作系统提供运行时服务的固件。BIOS预安装在个人电脑的主板上，是个人电脑启动时加载的第一个软件。

bus：总线，一组并行的导线，能携带地址、数据和控制信号。

C

clocking methodology：时钟方法，用来确定数据相对于时钟何时稳定和有效的方法。

CISC和RISC：复杂指令集计算机和精简指令集计算机。

compiler：编译程序，将高级语言翻译成计算机所能识别的及其语言的程序。

D

data race：数据竞争，加入来自不同线程的两个访存请求执行同一个地址，它们连续出现，并且其中至少一个是写操作，那么这两个存储访问形成数据竞争。

datapath element：数据通路部件，一个用来操作或保存处理器中数据的单元。在MIPS实现中，数据通路部件包括指令存储器、数据存储器、寄存器堆、ALU和加法器。

DIMM：双列直插内存模块（Dual Inline Memory Module），使用240个引脚，以64位为块传送数据到内存控制器和从内存控制器传出数据。

DLL：动态链接库（dynamically linked ），在程序执行过程中才被链接的库例程。

DMA：直接内存访问（Direct Memory Access），设备自己执行读或者写总线事务而不需要CPU干涉。如CPU从磁盘读数据过程中磁盘控制器确定目标扇区并将信息传送给主存。

DoS攻击：拒绝服务攻击（denial-of-service attack），亦称洪水攻击，是一种网络攻击手法，其目的在于使目标电脑的网络或系统资源耗尽，使服务暂时中断或停止，导致其正常用户无法访问。使用网络上两个或以上被攻陷的电脑作为“僵尸”向特定的目标发动DoS攻击时，称为分布式拒绝服务攻击（distributed DoS attack，简称DDoS攻击）。

E

edge-triggered clocking：边沿触发的时钟，一种所有的状态发生于时钟跳变的边沿的时钟机制。

end-end principle：端到端原则，某种功能必须基于端到端实现，与在较高级别提供这些功能的代价相比，在较低级别上设置的功能可能使冗余的或几乎没有价值的。

F

finite-state machine：有限状态机，由一组输入和输出、以及下一状态函数和输出函数组成的时序逻辑函数。下一状态函数将当前状态和当前输入映射为一个新的状态，输出函数将当前状态和当前输入映射为一组确定的输出。

firmware：固件，是一种嵌入在硬件设备中的软件，在系统通电时运行。通常它是位于特殊应用集成电路（ASIC）或可编程逻辑器件（PLD）之中的闪存或EEPROM或PROM里，有的可以让用户更新。可以应用在非常广泛的电子产品中，从遥控器、计算器到电脑中的键盘、硬盘，甚至工业机器人中都可见到它的身影。

flash memory：闪存，一类非易失性存储器，基于EEPROM，目前基于闪存的固态硬盘SSD大受欢迎。

forwarding：前推，也叫旁路（bypassing）。一种解决数据冒险的方式，具体做法是从内部寄存器而非程序员可见的寄存器或存储器中提前取出数据。

G

GCC：GNU编译器套装（GNU Compiler Collection），GNU计划的关键部分，也是GNU工具链的主要组成部分之一。GCC（特别是其中的C语言编译器）也常被认为是跨平台编译器的事实标准。

H

handle：句柄，在程序设计中，句柄是Windows操作系统用来标识被应用程序所创建或使用的对象的整数。其本质相当于带有引用计数的智能指针。当一个应用程序要引用其他系统（如数据库、操作系统）所管理的内存块或对象时，可以使用句柄。

I

ILP：指令级并行（instruction-level parallelism），指令间的并行性。

interrupt vector：中断向量，中断服务程序的入口地址，或中断向量表（它是一个中断处理程序地址的数组）的表项。系统必须维护一份中断向量表，每一个表项记录一个中断处理程序（ISR，Interrupt Service Routine）的地址。当外部事件或异常产生时，由硬件负责产生一个中断标记，CPU根据中断标记获得相应中断的中断向量号，然后由CPU根据中断向量表的地址和中断向量号去查找中断向量表获得相应中断号的中断程序地址，进一步执行对应的中断处理程序。

ISA：指令集架构（Instruction Set Architecture），用以定义机器机程序的格式和行为，定义了处理器状态、指令的格式，以及每条指令对状态的影响。采用《计算机组成与设计》上的定义，其包含了编写正确运行的机器语言程序所需要的全部信息，包括指令寄存器、存储访问和I/O等。

issue packet：发射包，在一个时钟周期内发射的多条指令的集合。这个包可以由编译器静态生成，也可以由处理器动态生成。

J

JIT编译器：即时（just in time）编译器，一类通用编译器的名称，编译器能够在运行时将解释的代码段翻译成宿主计算机上的机器语言。

L

LCD：液晶显示（Liquid Crystal Display），其原理为控制光的传输。

linker：链接器，也叫链接编辑器（link editor）。它是一个系统程序，把各个独立汇编的机器语言程序组合起来并且解决所有未定义的标记，最后生成可执行文件。

loader：加载器，把目标程序装载到内存中以准备运行的系统程序。

loop unrolling：循环展开，一种从存取数组的循环中获取更多性能的技术，其中循环体会被复制多份并且不同循环体中的指令可能会调度在一起。

M

MIME：多用途的网络邮件扩充协议（Multipurposse Internet Mail Extensions），该协议包含了多种文件类型。如text/html、text/plain、image/png等。

MIPS：具有两种含义，1）每秒百万条指令（million instructions per second）；2）Microprocessor without Interlocked Pipeline Stages，起初为一个研究项目，后来发展为一种指令集架构。

multiple issue：多发射，一个单时钟周期内发射多条指令的机制。

O

objected oriented language：面向对象语言，一种针对对象而不是动作的编程语言，或者针对数据而不是逻辑的编程语言。

operating system：操作系统，为了使程序更好的在计算机上运行而管理计算机资源的监控程序。

out-of-order execution 和 in-order commit：乱序执行和顺序提交。前者指“流水线执行的一种情况，即执行的指令被阻塞时不会导致后面的指令等待”；后者指“流水线执行的结果以取指顺序写回程序员可见存储器的一种提交方式”。

P

page table：页表，保存着虚拟地址和物理地址之间转换关系的表。页表保存在主存中，通常使用虚页号来索引，如果这个虚页当前在主存中，页表中的对应项将包含虚页对应的物理页号。

PC-relative addressing：PC相对寻址，一种寻址方式，将PC和指令中的常数相加作为寻址结果。

PIC：位置无关代码（Position-Independent Code，PIC），可以加载而无需重定位的代码。

procedure：过程，根据提供的参数执行一定任务的存储的子程序。

prompt: 提示，提示符

Posix：可移植操作系统接口（Portable Operating System Interface），是IEEE为要在各种UNIX操作系统上运行软件，而定义API的一系列互联关联的标准的总成，其正式称呼为IEEE Std 1003，而国际标准名称为ISO/IEC 9945。

pseudoinstruction：伪指令，汇编语言指令的一个变种，通常被看作一条汇编指令。

R

RAM：随机访问存储器，Random-Access Memory，分为静态的和动态的，即SRAM和DRAM，静态比动态更快、更贵。

register file：寄存器堆，包含一系列寄存器的状态单元，可以通过提供寄存器号进行读写。

response time：响应时间，也叫执行时间（execution time），是计算机完成某任务所需的总时间，包括硬盘访问、内存访问、I/O活动、操作系统开销和CPU执行时间等。

ROM：只读存储器，Read-Only Memory，其中有的类型可读也可写，其通过能够被重编程的次数以及重编程机制进行区分。

S

SIMD：单指令多数据（Single instruction Multiple Data）。通过SIMD并行可以同时对一组数据执行相同的操作从而实现空间上的并行。

single-cycle implementation：单周期实现，也称为单时钟周期实现，即一个时钟周期执行一条指令的实现机制。易于理解但是现实中因为太慢不实用。

SPEC：system performance evaluation cooperative，由许多计算机销售商共同出资赞助并指出的合作组织，目的是为现代计算机系统建立基准测试程序集。

SSD：一种基于闪存的存储技术，一个SSD封装由一个或多个闪存芯片和闪存翻译层组成，闪存芯片与传统磁盘中由多个盘片封装成的机械驱动器对应，硬件/固件构成的翻译层则与磁盘控制器对应。

SSE指令：流SIMD（Streaming SIMD Extensions）。SSE历经几代，已经更新为AVX（高级向量扩展，advanced vector extension），可以通过一条指令产生对多个数据值的计算，这些值存储在一组特殊的向量寄存器中。

straight-line code：无分支结构（如while; do-while; for）的代码。

symbol table：一个用来匹配标记名和指令所在内存字的地址的列表。

system call：系统调用，应用程序通过系统调用指令将处理器的控制权交由操作系统内核，内核执行被请求的操作后将结果返回给应用程序。

systems software：系统软件，指提供常用服务的软件，包括操作系统、编译程序、加载程序和汇编程序等。

T

throughput：吞吐率，也叫带宽（bandwidth），性能的另一种度量参数，表示单位时间内完成的任务数量。

TLB：翻译后备缓冲器（Translation Lookaside Buffer），一个小的、虚拟寻址缓存，包含在内存管理单元MMU中，每一行都保存着由单个PTE组成的块，通常有较高的相联度。（另一种定义是，用于记录最近使用地址的映射信息的高速缓存，从而可以避免每次都要访问页表）

U

USB：通用串行总线（Universal Serial Bus），USB控制器是一个连接到USB总线的设备的中转机构，用以连接各种外围I/O设备。

V

VLIM：超长指令字（Very Long Instruction Word），一类可以同时启动多个操作的指令集，其中操作在单个指令中相互独立，并且一般都有独立的操作码字段。

W

Web cache：Web缓存，也叫代理服务器（proxy server），能够代表初始Web服务器满足HTTP请求的网络实体。

workload：工作负载，运行在计算机上的一组程序，可以直接使用用户的一组实际应用程序，也可以从实际程序中构建。一个典型的工作负载必须指明程序和相应的频率。

write back：写回，当发生写操作时，新值仅仅被写入cache块中，只有当修改过的块被替换时才需要写到较低层存储结构中。

write buffer：写缓冲，一个保存等待写入主存数据的缓冲队列。

write-through：写直达，也译为写通过或写穿。写操作总是同时更新cache和下一存储器层次，以保持二者一致性。

WSC：仓储规模计算机（Ware-house Scale computer），巨型的数据中心，为云计算的硬件基础。

WWW：万维网（World Wide Web），也称为Web，是一个透过互联网访问的，由许多互相链接的超文本组成的系统。它不等同于互联网（Internet，即因特网），只是互联网所能提供的服务之一，是靠着互联网运行的一项服务。
