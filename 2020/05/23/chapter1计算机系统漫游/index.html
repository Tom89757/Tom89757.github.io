1.1 信息就是位+上下文

以一个简单的hello.c程序为例，里面的每个字符都以ASCII码的形式存储，每个字符占据1 byte=8 bits

1.2 程序被其他程序翻译成不同的方式

一个程序的生命周期往往以高级语言写成的程序开始，以一个二进制的机器可执行文件结束。以hello.c为例，首先由预处理器cpp识别第一行，将系统头文件stdio.h中的内容直接插入程序得到hello.i文件；再通过编译器ccl将hello.i转为由汇编语言表示的hello.s（由于汇编语言是通用的，所以同一机器上可以运行多种高级语言）；接下来通过汇编器as将hello.s转为二进制的可重定位目标程序hello.o（此处可将重定位理解为在地址空间中可定位程序中机器指令所在的位置）；最后还需通过链接器ld将printf函数对应的已编译好的printf.o文件链接到hello.o供其调用，即得到最终的二进制的可执行目标文件hello，其可加载到内存中供系统执行。

1.3 了解编译系统是如何工作是大有益处的

  • 优化程序性能：我们可以通过比较相近的实现相同功能的语句间的性能差异
  • 理解链接时发生的错误：链接器经常会报告无法解析一个引用，引用可能是函数或者变量，对这些函数或者变量我们应该采取怎样的存储模式和顺序。
  • 理解缓冲区溢出错误：安全区漏洞经常由缓冲区溢出错误引起，编译系统的学习可以帮助我们理解堆栈原理和缓冲区溢出错误

1.4 处理器读取并解释存储在内存中的指令

shelll为一个命令解释器，会输出一个提示符等待输入，往往使用回车表示输入完成，然后shell会判断输入的第一个单词是否为内置的shell命令，如果不是他会假定其为一个可执行文件的名字，并从内存中加载并执行。

系统的硬件组成

  • 总线：用以传递信息的电子通路，通常以字节块传递信息，字节块可简称为字，一般为4或8个字节，分别对应32位和64位。待解决的问题为IO/系统/内存总线的区别和联系
  • I/O设备：如鼠标/键盘/显示器/磁盘，其中鼠标/键盘/磁盘通过控制器与IO总线连接，显示器通过适配器与IO总线连接，控制器为设备自身或者主板上的芯片组，适配器则为插在主板上的卡，个人认为二者的主要区别在于其宏观上的可移植性。
  • 主存：用于存储程序和数据的临时存储设备。物理上由一系列的DRAM芯片组成，逻辑上为一个字节数组，每个字节有对应的地址，地址由零开始。
  • CPU：也称处理器。其核心为程序计数器PC，PC本质上为一个字大小的存储器／寄存器，其指向当前机器指令对应的主存地址；其他部分为ALU／寄存器文件。根据指令CPU可执行加载/存储/操作/跳转等操作。（需要区分的是处理器的指令集架构和微体系结构mark，前者为宏观表现即指令运行的效果，故比较直观简单，后者为具体实现，较为复杂，后续章节会进一步介绍）

运行hello程序

在shell窗口敲下hello过程中，hello通过IO总线和系统总线被一一读入寄存器，继而存储到主存中；当敲下回车键时，意味着输入结束，主存通过直接存储器存取技术而不用通过处理器直接从磁盘中加载hello程序和数据；加载完成后，处理器依次执行程序中的机器指令，并将输出结果输出到显示器上。

了解整个流程后，需要追问一个问题，shell究竟是什么。前面说到它是一个命令行解释器，属于软件，这里似乎要用到计算机的多层结构，mark一下留待解决（后面操作系统管理硬件的部分做了简单介绍）。

1.5 高速缓存至关重要

在处理器和主存之间插入更小的存储设备——高速缓存已经成为主流，并且高速缓存目前还采用了L1/L2/L3分级，以本人电脑为例，L1对应64KB，L2对应512KB，L3对应4MB，他们是通过静态随机访问存储器SRAMmark的硬件技术实现，和主存不同。（目前看来，提高处理器寄存器速度比提高主存速度要容易得多）

1.6 存储设备形成层次结构

目前存储器容量由小到大、速度由大到小依次为：寄存器、L1高速缓存、L2高速缓存、L3高速缓存、主存、本地二级存储（本地磁盘）、远程二级存储（分布式文件系统、Web服务器），都可将上一级看作下一级的高速缓存，数量级依次为几百B、几十KB、几百KB、几MB、几GB、几百GB／几TB、几百／几千TB。

1.7 操作系统管理硬件

操作系统可看作应用程序和硬件之间插入的软件，操作系统和应用程序共同构成了软件部分，处理器/IO设备/主存则共同构成了硬件部分。

操作系统需要实现两个基本功能mark：

  • 防止硬件被失控的应用程序滥用
  • 通过简单单一的机制控制复杂多样的硬件设备。

为实现上述功能，操作系统使用了三个重要的抽象概念：进程、虚拟内存和文件。

  • 进程是处理器/主存/IO设备三者的抽象表示，它营造了一种程序独立的占用处理器、主存和IO设备的假象。实际生活中，多个进程可以并发运行，表面上实现了多个程序的同时运行，实际上是通过上下文切换的微观机制来实现，任意时刻实际上都只有一个程序被执行。在进行两个进程间的切换时，操作系统会保存前一个进程的所有状态信息，即上下文，其包含了能够恢复该进程当前状态的所有信息，同时生成新的进程的上下文，从而实现多个进程间的切换。

    操作系统内核长期储存在主存之中，他是系统管理自身的所有进程的代码和数据结构的总和。

  • 线程。一个进程可由多个称为线程的执行单元构成，每个线程共享代码和全局数据。和进程相比，多线程间文件数据更为快捷。mark

  • 虚拟内存是主存/IO设备二者的抽象表示。进程的虚拟内存空间由5个部分组成，由下面区域到上面区域依次是：代码和数据，大小固定；malloc或者free创建的可伸缩的堆mark；共享库；用户栈，可伸缩mark；最顶部的内核虚拟内存。

  • 文件则是IO设备的抽象表示。文件即为字节序列，它为应用程序提供了同一的视图mark

该部分只简单的介绍了操作系统中涉及的概念，并未涉及具体调控机制，待后续详述

1.8 系统之间利用网络通信

此处将网络认为是一个IO设备，细想是合理的。网络即意味着交流，单次的交流即意味着一方输入、一方输出。

以telnet本地客户端与远程服务端间的网络为例，当使用客户端在登录远程服务端后，远程服务端等待本地客户端的输入，输入hello后，客户端将hello字符串发送至服务端，服务端shell执行hello程序并得到输出结果，然后服务端将输出结果发送至客户端，并在客户端的显示器上打印出来。

1.9 重要概念

Amdahl定律

简单但具有洞察力的定律：

其中$\alpha$为某部分所花费的时间占比，$k$为该部分效率提升的的比例。当$\alpha=0.6$、$k=3$时，比值为$\frac{5}{3}$，和$k$相差较多，可见只有当占比足够大的一部分速度提升时，才能显著提升整个系统的速度。当$k$趋向于$+\infty$时，比值为$\frac{1}{1-\alpha}$，这意味着$\alpha$较小时，性能的提升极其有限。

并发和并行

并发是指一个系统可以同时进行多种活动，而并行则是通过并发使一个系统运行得更快。

  • 线程级并发。目前已由单处理器单线程构成的系统发展为多处理器超线程构成的系统，但他们都是通过单一的操作系统控制。单处理器系统通过进程间切换模拟出的宏观上的并发，而多处理器则通过多核实现真正物理层面的并发。根据前面所提到的存储结构，每个核都有L1和L2高速缓存，L1由数据高速缓存和指令高速缓存构成，其速度仅次于寄存器并与寄存器进行信息传输，L2则是每个核统一的高速缓存，为L1的下一级缓存，然后每个核的L2又共用一个同一的高速缓存L3，L3与主存进行信息传输。

    超线程这一架构的使用是为了更充分的利用处理器的处理资源，往往表现为一部分如PC和寄存器文件具有多个备份，而另一部分如ALU则只有一份，从而使得一个线程在进行诸如数据的加载时执行另一个线程。目前主流的架构为双线程，可能的解释为：使用双线程可以更好的平衡单线程牺牲掉的处理资源和多线程架构的复杂化两者的关系，达到最优解，即既不至于浪费处理资源，又不至于由于架构的复杂化产生太多管理成本。

    值得一提的是，多处理器不仅满足了用户处理多个任务的需要，而且使用多线程方式编写的程序mark也能得到更快的运行。

  • 指令集并行。这个概念是指现代的单个处理器可以同时处理多条机器指令，后续会谈到流水线设计mark和超标量处理器（在少于一个时钟周期的时间内执行一条指令）mark。

  • 单指令、多数据并行。即SIMD并行mark，Single Instrucition Multiple Data，允许一个指令产生多个可以并行执行的操作。

计算机系统中抽象的重要性

抽象在计算机科学中无处不在。将函数抽象为应用程序接口API，将程序抽象为进程，将程序存储器抽象为虚拟内存，将I/O设备抽象为文件。我们还可以将操作系统、处理器、主存和I/O设备构成的整个系统抽象为虚拟机mark。

至此，第一章的内容全部结束。回顾一下，它展示了一个C语言程序的执行过程、网络通信、进程／虚拟内存／文件的抽象，从而将一系列的概念联系起来，并指出了几个重要的设计和理念——高速缓存、Aｍｄａｈｌ定律、并发和并行、抽象．
