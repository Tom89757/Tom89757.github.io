1.1 什么是因特网

可以从两个角度回答这个问题：

  1. 描述因特网的具体组成，即构成因特网的基本硬件和软件组件。
  2. 为分布式应用提供服务的联网基础设施。

具体构成描述

随着各种非传统设备（如手机、平板、手表、汽车等）连接到因特网，计算机网络的名称有些过时。连接到因特网的设备称为主机（host）或者端系统（end system），如下图所示：

端系统通过通信链路（communication link）和分组交换机（packet switch）连接到一起。需要掌握分组（packet）和路径（route）的概念，前者指发送端系统形成的信息包，后者指从发送端到接收端所经过的一系列通信链路和分组交换机。因特网中最著名的分组交换机是路由器（router）和链路层交换机（link-layer switch）。

端系统通过ISP（Internet Service Provider，因特网服务提供商）接入因特网，每个ISP本身就是由多台分组交换机和多段通信链路组成的网络。较低层ISP通过较高层ISP（如Level 3 Communication、AT&T、Sprint和NTT）互联，较高层ISP由通过高速光纤链路互联的高速路由器组成。

端系统和分组交换机等因特网部件运行一系列协议，这些协议控制因特网中信息的接收和发送。TCP/IP协议是其中最重要的两个协议，IP协议定义了在路由器和端系统之间发送和接收的分组格式。协议意味着达成一致，即标准。因特网标准由IETF研发，其标准文档称为RFC（Request For Comment），定义了TCP、IP、HTTP和SMTP等协议，目前RFC已达7000个；其他网络组织也在指定用于网络组件的标准，最重要的是针对网络链路的标准，如IEEE制定的以太网和无线WiFi的标准。

服务描述

当前因特网所服务的应用程序涉及多个相互交换数据的端系统，故这些应用程序又称为分布式应用程序（distributed application）。

端系统上分布式应用程序之间对相互之间发送数据的需求引出了套接字接口（socket interface）的概念，其规定了应用程序之间请求因特网基础设施来向对方交付数据的方式，具体来说套接字接口是发送数据的应用程序必须遵循的一套规则集合。

以上只是从两个角度说明了因特网的一些属性和功能，后续章节将具体介绍相关术语和工作原理。

广域无线接入：3G和LTE

智能手机对网络的要求更为严苛，它们要求网络信号更大面积的覆盖，这些设备应用了与蜂窝移动电话相同的无线基础设施，通过蜂窝网提供商运营的基站来发送和接收分组。3G无线技术已经逐渐淘汰，4G已经成为主流，5G也已经开始部署，甚至6G的研发已经提上了日程。后续会对WiFi、3G和LTE等技术作更详细的介绍。

物理媒体

对一个比特来说，其从一个端系统开始传输，通过一系列链路和路由器，到达另一个端系统，其间通过了一系列“发射器-接收器”对，每个对之间该比特的传输都通过跨越一种物理媒体传播电磁波或光脉冲来实现。

物理媒体（physical medium）分为导引型媒体和非导引型媒体两类，前者电波沿固体媒体前行，如光缆、双绞铜线和同轴电缆；后者点播在空气或外层空间中传播，如无线局域网或数字卫星频道。下面依次介绍各种物理媒体：

  1. 双绞铜线：两根绝缘铜线组成，用于局域网，速率可达10Gbps，目前是高速LAN联网的主导性解决方案。
  2. 同轴电缆：两个铜导体组成，两导体同心而非并行，在电缆电视系统中相当普遍。
  3. 光纤：细而柔软的、能够导引光脉冲的媒体，速率可达数百Gbps。其不受电磁干扰，长传输距离后信号衰减少，可防窃听。目前是跨海链路的传输媒体，但短途传输较少使用，因为成本较高。
  4. 陆地无线电信道：无线电信道承载电磁频谱中的信号，不需要安装物理线路，根据传输距离分为三类：短距离；局域；广域，分别对应蓝牙耳机等设备、无线局域网、蜂窝接入技术。
  5. 卫星无线电信道：以数百Mbps速率运行的卫星链路通常用于那些无法使用DSL或电缆因特网接入的区域。

1.3 网络核心

上一节考察了网络边缘，本节试图去接近网络核心，即由互联网端系统的分组交换机和链路构成的网状网络。下图中粗线部分为网络核心：

分组交换

端系统彼此交换报文，报文包含协议设计者需要的任何东西。为了从源端系统向目的端系统发送一个报文，源将长报文划分为较小的数据块，称之为分组，每个分组都通过通信链路和分组交换机传送（如前面所述，分组交换机主要为路由器和链路层交换机）。

存储转发传输

多数分组交换机在链路的输入端使用存储转发传输，即交换机在接收到整个分组后，才能向输出链路传送该分组。链路传输速率为R时，P个大小为L的分组，通过N条速率均为R的链路到达目的地的时延为：$d_{端到端} = (N+P)\frac{L}{R}$。下图展示了一个交换机、两条链路的情况：

转发表和路由选择协议

上面已经描述了如何存储和缓存分组，下面将讨论路由器如何决定向哪条链路转发分组。

实际上，源在分组的首部包含了目的地的IP地址，路由器中具有一个转发表用于将分组中的目的地址（每个分支处的选择通常只使用部分地址）映射为输出链路。

可察觉到，上述过程的关键在于转发表，转发表由路由选择协议自动设置，该协议可以决定从每台路由器到每个目的地的最短路径。书中友好的提供了 路由网站 来展示某个源到本机的路径。

电路交换

通过网络链路和交换机移动数据有两种基本方法：电路交换（circuite switching）和分组交换（packet switching）。本小节讨论电路交换网络。

电路交换网络与分组的区别在于，它在通信期间预留了沿路径通信所需资源（缓存，链路传输速率）。传统的电话网络即是电路交换网络，它为发送方-接收方的连接预留了带宽，发送方能够以确保的恒定速率向接收方传送数据。下图为一个电路交换网络：

图中两台主机间的端到端电路交换连接的传输速率为1Mbps的四分之一，即250kbps。

电路交换网络中的复用

链路中的电路通过频分复用或者时分复用来实现：

  1. 频分复用：FDM，Frequency-Division Multiplexing，链路的频谱由跨越链路创建的所有连接共享，在连接期间链路为每个连接专用一个频段，频段的宽度称为带宽。如调频无线电台使用FDM共享88MHz到108MHz的频谱，每个电台被分配一个特定频段。
  2. 时分复用：TDM，Time-Division Multiplexing，时间被划分为固定期间的帧，每个帧被划分为固定数量的时隙，网络在每个帧中为每个连接指定一个时隙。

如下图所示：

分组交换的拥趸辩驳道，电路交换不仅造成资源的浪费，而且创建端到端电路和预留端到端带宽是复杂的。

分组交换和电路交换的对比

诚然，分组交换不适合实时服务（如视频会议和电话），但是同样速率的链路，分组交换可以同时为更多的用户提供服务，也能使得某个用户（其他用户静默而不产生分组）突然产生的大带宽需求得到满足。

分组交换按需分配链路资源的特性使得它是发展的趋势，电路交换网络也在向分组交换网络迁移。（海外电话由于较为昂贵，已经使用了分组交换网络）

网络的网络

前面介绍了端用户和内容提供商接入ISP的技术，如DSL、电缆、FTTH、WiFi和蜂窝等，但这只解决了连接问题中的小部分，因为因特网由数以亿计的用户构成。要实现如此大规模的连接，ISP自身必须互联，即创建“网络的网络”。

“网络的网络”结构的形成由经济和国家策略驱动，而非由性能考虑驱动，这意味着在这个结构中自发形成了多层ISP。第一层ISP类似我们假想的全球传输ISP，如Level 3 Communication、AT&T、Sprint和NTT；低一层的ISP是区域ISP，每个接入ISP向其连接的区域ISP支付费用。在现实中，这个等级结构的层数可能更多，每一层都存在客户－提供商关系。这个结构在书中称为网络结构3。

但网络结构3并不能精确反应当今的因特网，还需要在该结构上增加存在点（Point of Presence，PoP）、多宿、对等和因特网交换点。这些概念是对相邻层次和同层次的ISP之间关系的补充，书上有简要介绍，此处不再赘述。由此衍生出了网络结构4——由接入ISP、区域ISP、第一层ISP、PoP、多宿、对等和IXP（因特网交换点，Internet Exchange Point）组成。

当今的互联网为网络结构5，它通过在网络结构4顶部增加内容提供商网络（content provider network，如谷歌）构建而成。如下图所示：

谷歌数据中心经过专用的TCP/IP网络互联，该网络跨越全球，且与公共因特网独立。谷歌专用网络既可以通过第一层ISP向端用户传输数据，也可以直接或通过IXP与之连接，从而减少了向第一层ISP支付的费用，同时加强了对其服务如何交付给端用户的控制。

内容提供商创建自己的专用网络已经成为当今因特网的潮流。

1.4 分组交换网中的时延、丢包和吞吐量

理想情况下，我们希望因特网服务能够在任意两个端系统之间随心所欲地瞬间移动数据而没有任何数据丢失；现实中，我们需要限制端系统之间的吞吐量，时延、丢包不可避免。让人不由感叹，理想与现实的差距需要多少人投入多少精力去填补。

分组交换网中的时延概述

如前面所述，分组从一台源主机出发，经由一系列路由器，在目的主机中结束其历程。在该过程中，分组经受了不同类型的时延：节点处理时延、排队时延、传输时延和传播时延，它们累加起来即为总时延。

时延的类型

如下图所示：

有四种类型的时延：

  1. 处理时延：其包括检查分组首部和决定将该分组导向何处所需要的时间、检查比特级别的差错所需要的时间等。
  2. 排队时延：其取决于先期到达的正在排队等待向链路传输的分组数量，队列为空时时延为0。
  3. 传输时延：传输时延为$L/R$，即将该分组的所有比特推向链路所需要的时间。
  4. 传播时延：一个比特从链路起点到下一个路由器传播所需的时间。传播速率取决于物理媒体（双绞铜线、光纤等），速率范围为$2 10^8 \sim 3 10^8$。传播时延为$\frac{d}{s}$，其中d为传播距离，s为传播速率。
  5. 传输时延和传播时延的比较：上述描述已经十分明晰。

排队时延是节点总时延中最复杂和有趣的部分。排队时延取决于流量到达该队列的速率、链路的传输速率和到达流量的性质。书中对排队时延作了数学化的描述，由于其不是我们学习的重点，暂略mark。下图展示了平均排队时延和流量强度的关系：

丢包

现实中，一条链路前的队列只有有限的容量，其依赖于路由器涉及和成本。当流量强度大于1时，队列会逐渐占满，随后到达的分组会溢出，即发生丢包。时延和丢包概率都是衡量节点性能的重要因素，后续会讲述如何通过重传来避免丢包，确保所有数据从源传送到目的地。

端到端时延

前面的讨论集中在节点时延，现在考虑从源到目的地的总时延。假定源主机和目的主机间有N-1台路由器，没有排队时延，则端到端时延为：$d{end-end} = N(d{proc} + d{trans} + d{prop})$，其中$d_{trans}$为L/R。

Traceroute

Traceroute程序的具体细节此处略去描述。该程序可以重建分组从源到目的地所采用的路由，并且确定到所有中间路由器的往返时延。

端系统、应用程序和其他时延

除上述时延类型外，还有其他类型，书上未着过多笔墨。

计算机网络中的吞吐量

端到端吞吐量也是计算机网络性能的重要量度。

1.5 协议层次及其服务模型

目前已经见识了因特网的复杂，其包括：大量的应用程序和协议、各种类型的端系统、分组交换机以及各种类型的链路级媒体。下面将对其网络体系结构进行讨论。

分层的体系结构

书中以航空系统为例，展示了分层的体系结构对我们理解和控制复杂系统的巨大作用，如下图所示：

分层结构带来的模块化使得某层提供的服务实现易于改变，且这种改变不影响该系统其他层次的组件。

协议分层

网络设计者以分层（layer）的方式组织协议以及实现这些协议的网络硬件和软件。一个协议层能够用软件、硬件或两者的结合来实现，协议分层具有概念化和结构化的优点，但是可能冗余较低层的功能或者某一层依赖于其他层的信息，使系统复杂化或者背离层次分离的目标。

将上述的考虑综合起来，各层的所有协议被称为协议栈（protocol stack），后续将采用自顶向下（top-down approach）的描述顺序，依次介绍应用层、运输层、网络层、链路层、物理层。（这不是唯一的协议栈，ISO提出计算机网络围绕7层来组织）如下图所示：

  1. 应用层：网络应用程序及它们的应用层协议存留的地方，包括HTTP、SMTP、FTP、DNS等协议。应用层协议分布在多个端系统商，多个端系统上的应用程序之间通过协议交换信息分组——报文（message）。
  2. 运输层：运输层在应用程序端点之间传送应用层报文，包括TCP和UDP协议，这两种协议均能运输应用层报文，但细节上曾在差异。运输层的分组成为报文段（segment）。
  3. 网络层：网络层将称为数据报（datagram）的网络层分组从一台主机移动到另一台主机，包括IP、路由选择协议等协议。源主机中的运输层协议（TCP/UDP）向网络层递交运输层报文段和目的地址。
  4. 链路层：网络层将数据包下传给链路层，链路层沿路径传递给下一个节点，在该节点链路层又将数据包上传给网络层。如以太网、WiFi和电缆接入网的DOCSIS协议。链路层分组称为帧（frame）。
  5. 物理层：链路层的任务是将整个帧从一个网络元素移动到邻近的网络元素，物理层的任务是将该帧中的一个个比特从一个节点移动到下一个节点。该层协议与链路（双绞铜线、光纤等）的实际物理传输媒体有关。

OSI 模型

该模型由国际标准化组织ISO提出，包含7层：应用层、表示层、会话层、运输层、网络层、数据链路层和物理层。与上述因特网协议栈相比，增加了两层——表示层和会话层。

  • 表示层：使通信的应用程序能够解释交换数据的含义。包括数据压缩、数据加密和数据描述。
  • 会话层：提供了数据交换的定界和同步功能，包括建立检查点和恢复方案的方法。

实际上，表示层和会话层的功能都由应用程序开发者来处理和构建。

封装

数据从发送端系统的协议栈向下，沿着中间的链路层交换机和路由器的协议栈上上下下，然后向上到达接收端系统的协议栈。如下图所示：

如上图所示，路由器实现了协议栈的三层，而链路交换机只实现了两层，这意味着路由器实现了IP协议，可以识别IP地址，链路交换机则不能，但它能识别以太网地址。主机则能实现所有五层协议，主机位于网络边缘，故其复杂性最高。

数据的传输过程多次使用了封装（encapsulation）这一概念，如运输层报文段封装了应用层报文和其他附加信息，网络层数据报、链路层帧都是对上一层信息的封装。在每一层：一个分组具有两种类型的字段：首部字段和有效载荷字段（payload field），后者来自上一层分组。

1.6 面对攻击的网络

网络安全领域面临和解决的问题是：坏家伙如何攻击网络，我们如何防御或者设计新型体系结构免除其攻击。

下面我们将讨论这些坏家伙对网络进行攻击的手段：

  1. 经因特网将有害程序放入计算机：恶意软件经由网络感染计算机，然后删除文件、窃取隐私信息等。大多恶意软件能够自我复制（self-replicating），以病毒（virus）或蠕虫（worm）的形式扩散，前者需要某种形式的用户交互，后者则不需要。
  2. 攻击服务器和网络基础设施：其具体名称为拒绝服务攻击（Denial-of-Service (DoS) attack），即DoS攻击。大多数DoS攻击属于下列三种类型之一：
    • 弱点攻击：向一台目标主机运行的易受攻击的应用程序或操作系统发送制作精细的报文。
    • 带宽洪泛：攻击者向目标主机发送大量分组，导致目标的接入链路拥塞，合法分组无法到达服务器。（带宽洪泛大多使用分布式DoS（Distributed DoS），攻击者控制多个源并让每个源向目标猛烈发送流量，DDoS充分利用了数以千计的受害主机组成的僵尸网络，如下图所示）
    • 连接洪泛：攻击者在目标主机中创建大量的半开或全开TCP连接，使目标主机陷入困境。

  1. 坏家伙能够嗅探分组：无所不再的因特网接入使得在无线传输设备的附近放置一台被动的接收机，该接收机就在能得到传输的每个分组的副本，该副本包含了各种敏感信息。记录每个流经的分组副本的被动接收机称为分组嗅探器（packet sniffer），有线环境中也能部署。（书中介绍了一个使用分组嗅探软件的实验）
  2. 坏家伙能够伪装成你信任的人：将具有虚假源地址的分组注入因特网的能力称为IP哄骗（IP spoofing），为避免接收到这样的分组，需要采用端点鉴别技术，第8章会进行探讨。

因特网最初是基于“一群相互信任的用户连接到一个透明的网络上”这样的模型来建造，随着因特网接入数以百亿计的设备，陌生的用户和服务器之间发生数据传输 ，网络安全性问题也不可避免，在相互信任的用户之间的通信成为特例而不是规则。

1.7 计算机网络和因特网的历史

本节介绍了网络和因特网发展的历程，仅作了解。

分组交换的发展：1961到1972

出现了分组交换、第一个分组交换网络、第一个主机到主机协议——网络控制协议（NCP，即RFC 001）

专用网络和网络互联：1972到1980

分组交换网络相继问世，网络的网络（将网络连接到一起）开始萌芽。如今三个重要的因特网协议——TCP、UDP和IP，在70年代末在概念上已经完成。

无线电网络和用于有线共享广播网络的以太网协议都开始出现。

网络的激增：1980到1990

NCP协议被TCP/IP替代。法国Minitel项目启动，其目的是使数据网络进入每个家庭，该项目于1984年取得巨大成功。

因特网爆炸：20世纪90年代

万维网（World Wide Web）应用程序的出现，将因特网带入世界上数以百万计的家庭和商业中。Netscape和微软之间的浏览器之战最终以微软胜利告终。各种应用程序流行，包括以下4种：

  • 电子邮件，包括附件和Web可访问的电子邮件
  • Web，包括Web浏览和因特网商务
  • 即时讯息（instant messaging），具有联系人列表
  • MP3的对等（peer-to-peer）文件共享，由Napster开创

因特网初创公司数以百计，微软、思科、雅虎、e-Bay、谷歌和亚马逊等成最大赢家。
