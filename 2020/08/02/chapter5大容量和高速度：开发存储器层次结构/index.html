5.1 引言

这一章主要探讨如何帮助程序员构建一个容量无限大的虚拟快速存储器。（在CSAPP中已有专门的一章讲述存储器层次结构，故本章会看得较快）

一个好的类比是，就像你不会同时以相同的概率查阅图书馆中的每一本书那样，一个程序也不会同时以相同的概率访问它全部的代码或数据。从中可以归纳出局部性原理，分为时间局部性（temporal locality）和空间局部性（spatial locality）。

程序本身的局部性是相当自然的，如循环中一组指令的反复使用体现了时间局部性，指令顺序执行则体现了空间局部性。根据局部性原理我们自然地将计算机存储器组织为存储器层次结构（memory hierarchy），显然该结构遵循“越小越快”的伟大思想。存储器的层次结构意味着数据与程序也组织为层次化结构，离处理器越远的层次访问时间越长，并且这种访问只能在相邻层次间进行，这意味着存储器层次结构需要提供在相邻层次间（而不需要考虑更远层次的传输）进行数据传输的机制，这又体现了“使用抽象简化设计”的思想。

下图展示了存储器层次的基本结构和两层次间信息交换的最小单元——块：

书中依次介绍了命中和缺失、命中率和缺失率、命中时间和缺失代价等概念。为了提高存储器层次结构的性能，我们的目的是减少命中时间和缺失代价，同时提高命中率。下面对命中时间和缺失代价进行严格定义：

  • 命中时间：访问某存储器层次结构所需要的时间，包括了判断当前访问是命中还是雀巢所需的时间。
  • 缺失代价：将相应的块从底层存储器 替换到高层存储器所需的时间，包括访问块、将数据逐层传输、将数据插入发生缺失的层和将信息块传送给请求者的时间。

由于所有程序都花费大量时间访问存储器，因为存储系统必然成为评估机器性能的一个主要指标，这也是我们需要理解存储器层次结构的必要性所在。下图展示了存储器的层次结构：

一个好的存储器层次结构应该拥有高的命中率，这意味着整个存储器系统拥有接近最快（最高层）的访问速度和接近最大（最低层）的存储空间。

5.2 存储器技术

如CSAPP中所描述，高速缓存通过SRAM实现，主存则通过DRAM实现，它们在速度和成本上有区别，其本质在于DRAM每bit占用的存储空间较少，故等量的硅制造的DRAM的容量比SRAM大。另外两种为闪存和磁盘，分别对应个人移动设备和PC中的二级存储器。下图展示了四种存储器技术的速度和成本：

SRAM技术

一种组织成存储阵列结构的简单集成电路，通常具有一个读写端口。由于摩尔定律的推动，当今处理器芯片中集成了多层次的cache。SRAM不需要刷新，可理解为其存储信息更为稳定，其访问时间与周期时间非常接近。

DRAM技术

DRAm使用电容保存电荷的方式存储数据，因此不能长久的保存数据，故需要周期性刷新。此外，它存储每bit只使用一个晶体管，故存储密度比SRAM更高，这也是其成本更低的原因。此处暂不描述其实现细节mark。下图展示了DRAM的内部组织：

下图展示了DRAM存储器的发展：

闪存

一种电可擦除的可编程只读存储器，对闪存的写操作会使存储器受损，故其擦除次数有限。为应对该限制，闪存通过其自带的控制器用来将写操作从已经写入很多次的块中映射到写入次数较少的块中，从而使写操作尽量分散，这种技术称之为损耗均衡（wear leveling）。采用这种技术，个人移动设备很难达到闪存的写极限。

cache缺失处理

cache缺失由两部分共同完成：处理器控制单元以及一个进行初始化主存访问和重新填充cache的独立控制器。我们假定下面的讨论基于顺序执行处理器，当cache缺失时其被阻塞，下面是指令cache缺失时的处理步骤：

  1. 把PC的原始值（当前PC减去4）送到存储器中。
  2. 通知主存执行一次读操作，并等待主存访问完成。
  3. 写cache项，将从主存取回的数据写入cache中存放数据的部分，并将地址的高位（从ALU中得到）写入标记域，设置有效位。
  4. 重启指令执行第一步，重新取指，这次该指令在cache中。

数据访问时对cache的控制基本相同。上述讨论的为进行读操作时发生缺失的处理策略。

写操作处理

对于store指令来说，其将数据写入数据cache而不改变主存的内容，故指令完成时cache和主存不一致。要使得二者一致，最简单的方法是同时将数据写入cache和主存，这称之为写直达（write-through）。当发生写缺失时，从主存中取出对应块中的字，然后将其存储cache中。

显然，上述策略将花费大量时钟周期将数据写入主存，另一种策略是写缓冲（write buffer）：

当一个数据在等待写入主存时，先将其放入写缓冲中，指令继续执行；当写缓冲满后，处理器执行到一个写操作后必须停下来直到写缓冲中有一个空位置。这种策略并不是永远奏效，当存储器完成写操作的速度慢于处理器产生写操作的速度时，即指令序列中包含的写操作比例较大时，再多的缓冲器也没有用。

另一种策略称之为写回（write-back）：

当发生写操作时，新值仅仅被写入cache块中，只有当修改过的块被替换时才需要写到较低层存储结构中。书中P267讨论了写回的细节，此处略mark。

5.5 可信存储器层次

前面集中展示了如何提高存储器的性能，但是再高的性能如果存储器存储数据的可靠性不够也无济于事。提高可信性的最好方法是冗余技术。

失效的定义

系统有两种服务状态：服务实现和服务中断。失效导致从服务实现到服务中断的切换，从服务中断切换到服务实现则称为恢复。失效又可分为间歇性和永久性，据此引出两个新的术语：可靠性和可用性。

  • 可靠性：是一个系统或模块能够持续提高用户需求的服务的度量，即从开始使用到失效的时间间隔。其度量方式包括平均无故障时间（mean time to failure，MTTF）和年失效率（annual failure rate，AFR）。
  • 可用性：是指系统正常工作时间在连续两次服务中断间隔时间中所占的比例。服务中断使用维修平均时间（mean time to repair，MTTR）来度量，失效间隔平均时间（mean time between failure，MTBF）= MTTF + MTTR。可用性 = MTTF / (MTTF + MTTR)

值得补充的是，可信性无法度量，可以度量的是可靠性和可用性。显然我们可以通过减少MTTF或者MTTR来提高可用性，前者可以通过以下技术：

  1. 故障避免技术：通过合理构建系统避免故障出现
  2. 故障容忍技术：采用冗余措施，当发生故障时，通过冗余措施保证系统继续工作
  3. 故障预报技术：对故障进行预测，从而允许在器件失效前进行替换

后者则可以采用故障检测、诊断和修复的工具来减少故障维修花费的时间。

纠正一位错、检测两位错的汉明编码（SEC/DED）

在理解错误检测编码前，需要理解二进制数间的距离，即汉明距离。

汉明使用奇偶校验码进行错误检测，这意味着N + 1（奇偶校验位）位码字中1的个数永远为偶数，该设定只能检测出1位（其实是奇数位）的错误。此时码字间最小的汉明距离为2（如果一个字有1位发生翻转，则其校验位同样翻转，故与翻转前的码字距离为2）

为做到检错的同时可以纠错，汉明发明了汉明纠错码（Hamming Error Correction Code，ECC），此时码字间的最小汉明距离为3：

  1. 对数据部分从左到右由1开始一次编号。（这与通常从右到左编号相反）
  2. 将所有编号为2的整数次幂的位标记为奇偶校验位（1，2，4，8，16…）。
  3. 其他剩余位置用作数据位（位置3，5，6，7，9，10，11，12，13，14，15，…）。
  4. 奇偶校验位的位置决定了其对应的数据位。
  5. 设置奇偶校验位，对各组进行偶校验。

如下图所示：

汉明并没有止步于此，通过付出增加1位的代价，使码字的最小汉明距离变为4，可以在纠正1位错的同时检测2位错。其操作为：在计算出上述的汉明奇偶校验位后，最后计算出整个字的偶校验位，并放置在最低位。

纠1位错检2位错（SEC/DED，Single error correcting / Double error detection）的技术现在广泛应用于服务器的内存。方便的是，8个字节的数据块做SEC/DED时恰好只需要一个字节的额外开销，这也是为什么许多双列直插式存储模块宽度位72位。

书中计算了SEC所需要的位数，有$2^p \ge p + d + 1$，即 $p \ge \log (p + d + 1)$。如d = 8 时，有p = 4。

此外，书中提及，在大型系统中，出现多位错的概率和整个内存芯片出错的概率变大，仅使用奇偶校验位或者SEC/DED是不够的，需要使用一种称为chipkill的技术；在网络系统中可能出现突发型错误，此时需要使用循环冗余校验，具体细节略。

5.6 虚拟机

虚拟机一直是大型机中的重要组成部分，但在单用户计算机中被忽视，其最近受到关注是因为：

  1. 在现代计算机系统中，隔离和安全的重要性在增长。
  2. 标准操作系统在安全性和可靠性方面存在缺陷。
  3. 在多个不相关的用户间共享一台计算机，尤其在云计算中。
  4. 处理器速度大幅增长，使得虚拟机引起的开销降至可以接受的范围内。

以下的讨论基于（操作）系统虚拟机：

在一个传统的平台上，一个单独的操作系统拥有所有的硬件资源，但是通过虚拟机，多个操作系统共享硬件资源。为了更好地描述虚拟机技术，书中引出了虚拟机监视器（virtual machine monitor，VMM，也叫管理程序）和主机的概念。

虚拟机不仅可以提供保护功能，还有其他两个优势：

  1. 软件管理：虚拟机典型的调度包括，一些虚拟机运行旧的操作系统，多数虚拟机运行当前的操作系统，少数虚拟机用来测试下一代操作系统版本。
  2. 硬件管理：虚拟机允许独立的软件堆能在共享硬件的同时独立运行；也支持将正在运行的虚拟机移植到另一台计算机上。

处理器虚拟化的开销取决于需要由VMM进行模拟的指令数目一级模拟速度的快慢。

虚拟机监视器的必备条件

虚拟机监视器（VMM）给客户软件提供了一个软件接口，分开每个客户端的状态，并且需要将自己从客户端软件中（包括客户操作系统）隔离。

系统级虚拟机的基本必备条件如下：

  • 至少两个处理器模式，系统级和用户级
  • 特权级指令集合只能在系统模式下使用，如果在用户模式下执行将会产生trap中断；所有系统资源只能由这些指令控制。

指令集系统结构（缺乏）对虚拟机的支持

由于虚拟机只是近期才考虑应用于桌面系统和基于PC的服务器，所以大部分指令集在创建时都没有考虑虚拟化的影响。

保护和指令集系统结构

保护需要同时依赖于体系结构和操作系统。在过去，IBM的大型机硬件和VMM采用以下三个策略改善虚拟机的性能：

  1. 降低处理器虚拟化的开销
  2. 降低由虚拟化引起的中断开销
  3. 中断发生时交给相应的虚拟机，而不用调用VMM，从而降低开销

5.7 虚拟存储器

虚拟存储器是一种将主存用作辅助存储器高速缓存的技术，即将主存作为磁盘（辅助存储器）的高速缓存。构造虚拟存储器出与两个动机：

  1. 允许云计算在多个虚拟机之间有效而安全地共享存储器
  2. 消除一个小而受限的主存容量对程序设计造成的影响

目前第1条为主要动机，我们需要在虚拟机之间进行保护，确保每个程序只能对划分给它的那部分主存进行读写操作。虚拟存储器使得我们能更有效地共享处理器和主存，实现了程序地址空间到物理地址的转换，加强了各个程序地址空间之间的保护。

同时，虚拟存储器将程序员从“谨小慎微的分配硬件资源”的困境中解脱出来，虚拟存储器自动管理由主存（物理存储器）和硬盘（辅助存储器）组成的两级存储器层次结构。

出与不同的历史根源，虚拟存储器和cache使用不同的术语描述，尽管它们具有相似的工作原理。书中一次介绍了页、缺页、虚拟地址、物理地址等概念，用它们可以描述对主存的访问。下图展示了分页的虚拟寻址的存储器被映射到主存中，称为地址转换：

页缺失的昂贵代价是采用虚拟存储系统的原因，一次缺页将花费数百万个时钟周期。下面是设计虚拟存储系统时需要考虑的一些关键性因素：

  1. 为了弥补较长的访问时间，页应该足够大。
  2. 允许存储器中的页以全相联的方式放置以降低缺页率。
  3. 缺页可以用软件处理，以避免访问磁盘导致的巨大开销；软件可以使用更先进的算法选择替换页，尽量降低缺页率。
  4. 采用写回机制。

页的存放和查找

设计人员通过对页的放置进行优化来降低缺页频率。当采用全相联方式组织页时，缺页发生时操作系统可以选择任意页来进行替换。全相联映射的难点在于项的定位，为此在虚拟存储系统中使用一个索引存储器的表来定位页，该表称为页表（page table）。

每个程序都有它自己的页表，用来将程序的虚拟地址空间映射到主存中。为了指出页表在存储器中三位置，硬件包含一个指向页表首地址的页表寄存器（page table register）。我们假定页表存储在存储器中一个固定的连续区域内。

缺页故障

当正在访问的页表中某一项有效位为0时，就会发生缺失，此时操作系统获得控制权，控制的转移由异常机制完成。接着操作系统根据虚拟地址找到目标页在磁盘或者闪存中的位置，将其取回主存。

此外，由于无法获知存储器中某一页什么时候将被替换出去，操作系统在创建进程的时候会在闪存或磁盘上为进程中所有的页创建空间，称为交换区（swap space），同样有页表或者类似的数据结构记录每个虚拟页在磁盘上的位置。下图为页表结构：

关于写

访问cache时，可以使用写直达机制，但是需要一个写缓冲区来隐藏写延迟。但是由于访问磁盘需要数百万个时钟周期，所以不能再使用写缓冲的机制，因为缓冲区会很快填满。故虚拟存储器必须使用写回机制，对存储器中的页进行单独的写操作，并且再该页被替换出存储器时在被复制到磁盘中去。

加快地址转换：TLB

由于页表存放在主存中，因此程序每次访存至少需要两次：第一次访存先获得物理地址，第二次访存才获得数据。利用局部性原理，现代处理器都包含一个特殊的cache以追踪最近使用过的地址变换，即快表（Translation-Lookaside Buffer，TLB），准确翻译是地址变换高速缓存。下图展示了TLB的结构：

TLB的一些典型的值为：

  • TLB大小：16~512个项。
  • 块大小：1~2个页表项（通常为4到8个字节）
  • 命中时间：0.5~1个时钟周期。
  • 缺失代价：10~100个时钟周期。
  • 缺失率：0.01%~1%。

关于TLB相联度的选择多样。可以选择全相联的组织方式，此时缺失率较低，并且TLB本身很小，实现代价不大；也可以选择相联度低且容量大的TLB，因为TLB的缺失比缺页频繁得多，使用相联度高的策略进行替换时代价较大。

虚拟存储器的保护

虚拟存储器最重要的功能就是允许多个进程共享一个主存，同时为这些进程和操作系统提供存储保护。

为了使操作系统能保护虚拟存储系统，硬件至少提供下面三种基本能力：

  1. 支持至少两种模式，并指出当前运行的进程是用户进程还是操作系统进程。
  2. 提供一部分处理器的状态，这部分内容是用户进程可读不可写的。
  3. 提供能让处理器再用户态和管理态下相互切换的机制——系统调用和异常返回。

使用这些机制将页表保存在操作系统进程的地址空间中，既可以让操作系统更改页表，调度内存资源，又可以保护页表不被用户进程更改。

我们同样要防止一个进程读取另一个进程的数据，这就要求操作系统给进程分配的虚拟地址空间映射到不相交的物理页上；于此同时，我们同样希望操作系统协助一个用户进程对其它进程以受限的方式（如只读）共享信息。

处理TLB缺失和缺页

当TLB中没有一个表项能匹配虚拟地址时，TLB缺失就会发生，分为两种情况：

  1. 页在主存中，只需要创建缺失的TLB表项。
  2. 页不在主存中，需要将控制权进给操作系统来解决缺页。

此外，TLB缺失或者缺页必须在访存发生的同一个时钟周期的末尾被判定，因此下一个时钟周期就开始进行异常处理而不是正常的指令执行。若非如此，紧接着执行的load或者store指令将会改写寄存器。

一旦操作系统知道了引起缺页的虚拟地址，它必须完成一下三个步骤：

  1. 使用虚拟地址查找页表项，并在磁盘上找到被访问的页的位置。
  2. 选择替换一个物理页；如果被选中的页被修改过，需要在把新的虚拟页装入之前将这个物理页写回到磁盘上。

小结

虚拟存储器是管理主存和磁盘之间数据缓存的一级存储器层次，它允许单个程序在主存有限的范围内扩展地址空间，同时支持多个活跃的进程共享主存。磁盘的速度很慢，这意味着缺页的代价很高，以下是降低缺页代价的策略：

  1. 增大页的容量以便利用空间局部性降低缺失率。
  2. 使用页表实现的虚拟地址和物理地址之间的映射采用全相联的方式，这样虚拟页可以被放置到主存中的任何位置。
  3. 操作系统使用类似LRU和访问位之类的技术选择替换哪一页。

为了使进程地址空间收到保护以及防止页表被进程改写，只有操作系统具有更改页表的权限（操作系统进程运行时处于管理模式）；为了加快处理器对存储器的访问速度，在处理器中内置了TLB，其扮演了地址转换cache的角色。

cache、存储器和TLB都建立在一组共同的原理和策略基础上，下一节将讨论这个共同的架构。

问题4：写操作如何处理

处理写操作有两种选择：

  1. 写直达：信息被同时写到cache的块和存储器层次结构低一层的块（对cache来说是主存）中。
  2. 写回：信息仅仅写到cache的块，被改写的块只有在它被替换时才写回到主存。虚拟存储系统由于写直达所需时钟周期数过大只能使用写回。

3C：一种理解存储器层次结构行为的直观模型

该模型可以很好的洞察存储器层次结构中引起缺失的原因以及层次结构的变化对缺失的影响。在该模型中，所有的缺失被分为以下三类（3C模型：three Cs model）：

  1. 强制缺失（compulsory miss）：也称为冷启动缺失（cold-start miss），对从没有在cache中出现的块的第一次进行访问引起的缺失。
  2. 容量缺失（capacity miss）：由于cache容纳不了一个程序执行所需要的所有块而引起的cache缺失，当某些块被替换出去，随后再被调入时，将发生容量缺失。
  3. 冲突缺失（conflict miss）：也称为碰撞缺失（collision miss），在组相联或者直接映射的cache中，多个块竞争同一个组时引起的cache缺失。冲突缺失在同样大小的全相联cache中不存在。

5.10 并行与存储器层次结构：cache一致性

多核处理器可能会共享一个公共的地址空间，这意味着多个cache共享数据，这就可能带来cache一致性问题，如下图所示：

一致性的简单定义是：在一个存储器系统中读取任何一个数据项的返回结果总是最近写入的值。但显然应该追求更为准确精细的描述，书中列出了三条满足一致性所需的条件，此处不详述。其关键概念为写串行化，即保证对同一个地址的写操作在所有处理器看来都具有相同的顺序。

实现一致性的基本方案

在多处理器系统中，cache通过提供共享数据的迁移和复制来支持一致性：

  • 迁移：数据项可以移入本地cache并以透明的方式使用。
  • 复制：当共享数据被同时读取时，cache在本地对数据项作了备份。

许多多处理器通过引入被称之为cache一致性协议的硬件协议来维护cache一致性，其关键在于跟踪所有共享数据块的状态。最常用的cache协议是监听（snooping）协议。

监听协议

实现一致性的一种方法是：在处理器写数据之前，保证该处理能独占地访问该数据项。这种协议称为写无效协议（write invalidate protocol），因为它在执行写操作时令其他副本无效。

5.15 谬误与陷阱

  • 陷阱1：在写程序或编译器生成代码时忽略存储系统的行为。

前面介绍的排序算法和cache分块技术证明考虑存储系统行为可以很容易使得算法性能翻倍。

  • 陷阱2：在模拟cache的时候，忘记说明字节编址或者cache块大小。

不说明字节编址或者cache块大小时，我们无法确定某一字节被放置在哪个cache块中。

  • 陷阱3：对于共享cache，组相联度少于核的数量或者共享该cache的线程数。

当组相联度过低而不满足上述要求时，可能引发冲突缺失而导致严重的性能缺陷。

  • 陷阱4：用存储器平均访问时间来评估乱序处理器的存储器层次结构。

在乱序处理中，当发生cache缺失时，指令继续执行，此时无法通过存储器访问时间来评估存储系统；当然，缺失时阻塞是能够评估的。

  • 陷阱5：通过在未分段地址空间的顶部增加段来扩展地址空间。

这是上世纪可能使用的操作，年代久远，暂不理解mark。

  • 谬误1：实际的磁盘故障率和规格书中声明的一致。

实际上，根据研究评估，实际结果往往比规格书中声明的故障率高出3到6倍。

  • 谬误2：操作系统是调度磁盘访问的最好地方。

磁盘调度的读操作比操作系统调度的读操作花费的旋转周期可能更短。

  • 陷阱6：在不为虚拟化设计的指令集体系结构上实现虚拟机监视器。

不太懂，mark。

5.16 本章小结

本章的关键概念是局部性原理和多级cache。为充分发挥存储系统的性能，可以依赖于软件，一方面可以通过重新组织程序结构，另一方面是通过预取在数据块真正被访问之前将其读入cache从而降低缺失率。

存储器层次结构对系统性能的重要性不断增长，要对其予以重视。
