本章讨论链路层，将围绕两种截然不同类型的链路层信道：广播信道和点对点通信链路展开。

6.1 链路层概述

先学习一些有用的术语：

  • 节点（node）：运行链路层协议的任何设备，包括主机、路由器、交换机和WiFi接入点。
  • 链路（link）：沿着通信路径连接相邻节点的通信信道。

下图展示了无线主机和服务器之间的6个链路层跳：

链路层提供的服务

链路层协议能够提供的可能服务包括：

  1. 成帧（framing）：将网络层数据报用链路层帧封装起来。
  2. 链路接入：媒体访问控制（Medium Access Control，MAC）协议规定了帧在链路上传输的规则。
  3. 可靠交付：当链路层协议提供可靠交付服务时，它保证无差错地经链路层移动每个网络层数据报。
  4. 差错检测和纠正：由于没有必要转发一个有差错的数据报，所以许多链路层协议提供一种机制检测这样的比特差错。

链路层在何处实现

链路层的主体部分在网络适配器（network adapter）种实现，有时也称为网络接口卡（Network Interface Card，NIC）。位于网络适配器核心的时链路层控制器，其通常是一个实现了诸多链路层服务（成帧、链路接入、差错检测等）的专用芯片，这些服务许多用硬件实现，如以太网协议和WiFi协议。

在发送端，控制器取得了由协议栈较高层生成并存储在主机内存中的数据报，在链路层帧种封装该数据报，然后遵循链路接入协议将该帧传进通信链路中。在接收端，控制器接受了整个帧，抽取出网络层数据报（如果链路层执行差错检测，发送端在帧的首部设置差错检测比特，接收控制器执行差错检测）。

6.2 差错检测和纠正技术

比特级差错检测和纠正（bit-level error detection and correction），即对从一个节点发送到另一个物理上连接的邻近节点的链路层帧中的比特损伤进行检测和纠正，这是链路层提供的两种服务。下图展示了差错检测与纠正的场景：

我们使用差错检测和纠正（Error Detection and Correction，EDC）比特来增强数据D。要保护的数据不仅包括从网络层传递下来的数据报，也包括链路帧首部中的链路级的寻址信息、序号和其他字段。

下面讨论三种检测差错的技术：奇偶校验、检验和、循环冗余检测。

奇偶校验

在发送的信息D中附加一个比特。使用偶校验方案时，可以检测出奇数个比特差错。下图展示了单比特奇偶校验方案的二维一般化方案：

该方案可以识别发生的单个比特的差错并确定其位置进行纠正，同时也可以检测出校验比特本身的单个比特差错并确定位置进行纠正。

接收方检测和纠正差错的能力称为前向纠错（Forward Error Correction，FEC），其可以减少发送方重传的次数，并允许接收方立即纠正差错，故在实时网络应用中有广泛应用。

检验和

如在3.3节中讨论的那样，因特网检验和（Internet checksum）将数据的字节作为16比特的整数对待并求和，和的反码作为携带在报文段首部的检验和，接收方对接收的数据（包括检验和）的和求取反码，其结果全为1则说明没有差错。

检验和方法需要相对小的分组开销（TCP和UDP的检验和只用了16比特），但与后面的CRC相比，提供相对弱的差错保护。二者分别在运输层和链路层使用，其原因在于运输层通过软件实现，需要采用简单且快速的方案；而链路层则使用了较多的硬件，其差错检测可以在适配器中用专用的硬件实现，可以采用复杂的CRC操作且能较快速的完成。

循环冗余检测

循环冗余检测（Cyclic Redundancy Check，CRC）在当今计算机网络中广泛应用，也称为多项式编码（polynoial code）。书中描述了具体的CRC编码操作，此处略：

6.3 多路访问链路和协议

前面提到的两种网络链路类型之一的广播链路（broadcast link），能够让多个发送和接收节点都连接到相同的、单一的、共享的广播信道上。如何协调多个发送和接收节点对一个共享广播信道的访问，成了一个需要考虑的问题，这就是多路访问问题（multiple access problem）。

多路访问协议便是用来规范节点在它们共享的广播信道上的传输行为。多个节点同时传输帧时，所有节点同时接收多个帧，便会导致传输帧在接收方处碰撞，涉及此次碰撞的帧都丢失，广播信道被浪费，这是需要多路访问协议的原因之一。

链路层中已经实现了几十种多路访问协议，可以分为3种类型：信道划分协议（channel partitioning protocol）、随机接入协议（random access protocol）和轮流协议（taking-turns protocol）。对于速率为R bps 的广播信道，多路访问协议应该具有以下所希望的特性：

  1. 当仅有一个节点发送数据时，该节点具有R bps 的吞吐量。
  2. 当有M个节点发送数据时，每个节点一段时间的平均吞吐量为R/M bps。
  3. 协议是分散的，不会因某主节点故障而使整个系统崩溃。
  4. 协议是简单的，使实现不昂贵。

信道划分协议

时分多路复用（TDM）和频分多路复用（FDM）是两种能够用在所有共享信道节点之间划分广播信道带宽的技术。下图展示了一个4节点的TDM 与 FDM 的例子：

TDM消除了碰撞且非常公平，其缺点在于节点的传输速率被局限在R/N bps；节点必须总是等待它在传输序列中的轮次。FDM则将R bps信道划分为不同的频段，并把每个频段分配给N个节点中的一个，其优点和缺点与TDM类似。

第三种信道划分协议是码分多址（Code Divsion Multiple Access，CDMA），TDM和FDM为节点分配时隙和频段，CDMA则为不同节点分配一种不同的编码，每个节点用它唯一的编码对它发送的数据进行编码。

在随机接入协议中，一个传输节点总是以信道的全部速率 R bps 进行发送。下面描述两种最常用的随机接入协议：ALOHA协议和载波侦听多路访问（CSMA）协议（以太网是一种流行并广泛部署的CSMA协议）。

3.载波侦听多路访问（CSMA）

类似于有礼貌的人类谈话，CSMA遵循两个重要的规则：

轮流协议

前面讨论的信道划分协议具有多路访问协议的第二个理想特性（M个节点，R/M bps吞吐量）；随机接入协议则具有第一个特性（1个节点，R bps吞吐量）。本节讨论的轮流协议则企图同时实现两种特性。

比较重要的轮流协议有两种：

  1. 轮询协议（polling protocol）：主节点以循环的方式轮询每个节点，告诉它们能传输帧的最大数量。其消除了碰撞和空时隙，但引入了轮询时延，并且主节点的故障会造成整个过程的崩溃。
  2. 令牌传递协议（token-passing protocol）：一个称为令牌（token）的小的特殊帧在节点之间以某种固定的次序进行交换。其问题在于一个节点的故障或者忘记令牌的释放可能导致整个信道崩溃。

DOCSIS：用于电缆因特网接入的链路层协议

该协议是上面学习的3大类多路访问协议的实例。一个电缆接入网通常在电缆王头端将几千个住宅电缆调制解调器与一个电缆调制解调器端接系统（Cable Modem Termination System，CMTS）连接，数据经电缆服务接口规范（Data-Over-Cable Service Interface Specifications，DOCSIS）定义了电缆数据网络体系结构及其协议。下图展示了CMTS和电缆调制解调器之间的上行和下行信道：

链路层寻址和ARP

主机和路由器具有链路层地址。本节的目标是弄清网络层地址和链路层地址的用途和区别，并学习地址解析协议（Address Resolution Protocol，ARP），其提供了将IP地址转换为链路层地址的机制。

1.MAC地址

事实上，并不是主机和路由器具有链路层地址，而是它们的适配器（即网络接口）具有链路层地址。然而，交换机并不具有与它们的接口相关联的链路层地址。

其原因在于交换机透明地在主机与路由器之间承载数据报，主机或路由器不必明确地将帧寻址到其间的交换机接口，如下图所示，与局域网相连的每个接口都有一个唯一的MAC地址：

对于大多数局域网（包括以太网和802.11无线局域网）而言，MAC地址的长度为6个字节，共有$2^{48}$个可能的MAC地址。下面的讨论假设一块适配器的MAC地址不变，MAC地址空间由IEEE组织管理。

适配器的MAC地址具有扁平结构，到哪都不会改变；而IP地址则具有层次结构，移动后会重新分配。一台主机具有一个不会改变的MAC地址和一个会改变的网络层地址。

需要补充的是，为了使网络体系结构中各层次称为极为独立的构建模块，不同的层次需要有它们自己的寻址方案。目前，已经看到3种类型的地址：应用层的主机名、网络层的IP地址以及链路层的MAC地址。

以太网

以太网几乎占领着现今的有线局域网市场，是目前为止最为流行的有线局域网技术。以太网之于本地区域联网，类似于因特网之于全球联网。相较于其他技术，其流行的原因是部署早、不复杂、便宜、数据传输速率并不逊色。

从20世纪10年代到如今，以太网从总线拓扑发展为星形拓扑，再到（将集线器改为）交换机。

1.以太网帧结构

如下图所示：

以太网帧的6个字段分别为：

  • 数据字段（46到1500字节）：承载了IP数据报。
  • 目的地址（6字节）：包含目的适配器的MAC地址。
  • 源地址（6字节）
  • 类型字段（2字节）：类型字段允许以太网复用多种网络层协议，IP和其他网络层协议都有各自的、标准化的编号。该类型字段和网络层数据报中的协议字段、运输层报文段中的端口号字段相似，但是为了将某一层的某协议与上一层的协议结合起来。
  • CRC（4字节）：循环冗余检测字段的目的是使得接收适配器检测帧中是否引入了差错。
  • 前同步码（8字节）：以太网帧以一个8字节的前同步码字段开始。前7个字节用于同步接收适配器和发送适配器的时钟，最后1个字节用来提醒接收适配器数据即将到来。

以太网技术向网络层提供无连接服务，即没有出现TCP中的握手，而是与IP的第三层数据报服务和UDP的第四层无连接服务类似。

以太网技术向网络层提供不可靠服务，发送适配器并不知道它传输的帧是否到达了接收适配器并通过了CRC校验。当使用UDP时，目的主机上的应用会看到数据中的间隙，但使用TCP时，由于没有响应确认报文，会导致重传数据。

链路层交换机

交换机的任务是接收链路层帧并将它们转发到初链路。交换机自身对子网中的主机和路由器是透明地，主机和路由器并不知道某交换机会接收帧并将他转发到另一个节点。

1.交换机转发和过滤

过滤功能是：决定应该将一个帧转发到某个接口还是将其丢弃；转发是：决定一个帧应该被导向哪个接口，并将该帧移动到那些接口。交换机的过滤和转发借助于交换机表（switch table）完成，如下图所示：

交换机表包含某局域网上某些主机和路由器的但不必是全部的表项，每个表项包括一个MAC地址、通向该MAC地址的交换机接口、表项放置在表中的时间。基于MAC而不是IP对称转发分组需要进行明确区分。书中对到达交换机某个接口的帧所发生的三种情况进行了讨论，较为简单，此处略。

2.自学习

交换机表是自动、动态和自治地建立起来的，没有来自网络管理员或来自配置协议的任何干预。概括地说，交换机是自学习（self-learning）的。其通过如下方式实现：

  1. 交换机表初始为空。
  2. 对于在每个接口接收到的每个入帧，该交换机在其表中存储该帧源地址字段中的MAC地址、该帧到达的接口和当前时间。
  3. 如果在一段时间（老化期（aging time））后，交换机没有接收到以该地址作为源地址的帧，就在表中删除这个地址。

交换机是即插即用设备（plug-and-play device），因为它们不需要网络管理员或用户的干预。

3.链路层交换机的性质

不同于基于集线器的总线或星形拓扑那样的广播链路，链路层交换机具有如下优点：

  1. 消除碰撞：在使用交换机（不使用集线器）构建的局域网中，没有因碰撞而浪费的带宽。
  2. 异质的链路：交换机将链路彼此隔离，因此局域网中的不同链路能够以不同的速率运行并且能够在不同的媒体上运行。
  3. 管理：交换机易于进行网络管理。当一个适配器工作异常或一条缆线断开时，不会导致整个系统崩溃。并且交换机可以收集带宽使用的统计数据、碰撞率和流量类型，提供给网络管理者。

书中补充了一种被称之为交换机毒化（switch poisoning）的对抗交换机的攻击，此处略。

4.交换机和路由器比较

交换机时第二层的分组交换机，而路由器是第三层的分组交换机，虽然二者都使用了存储转发机制。正如4.4节中描述的，使用“匹配加动作”的现代交换机能够基于不同的帧、数据报和运输层首部字段，执行通用的分组转发（典型的为使用OpenFlow标准的交换机）。

先考虑交换机的优点和缺点：

  • 交换机优点：即插即用，具有相对高的分过滤和转发速率。
  • 交换机缺点：交换网络的活跃拓扑限制为一颗生成树；对广播风暴不提供保护措施。

在考虑路由器的优点和缺点：

  • 路由器优点：网络中存在冗余路径时，分组通常不会通过路由器循环；分组不会限制在一颗生成树上，允许以丰富的拓扑结构构建因特网；对第二层的广播风暴提供了防火墙保护。
  • 路由器缺点：非即插即用，需人为配置IP地址；对每个分组的处理时间比交换机更长，因为必须处理高达第三层的字段。

6.5 链路虚拟化：网络作为链路层

本节中，将考虑多协议标签交换（Multiprotocol Label Switching，MPLS）网络。与电路交换的电话网不同，MPLS是一种分组交换的虚电路网络，有自己的分组格式和转发行为。

多协议标签交换

该技术用于改善IP路由器的转发速度。它采用固定长度标签的概念，其目标是：对于基于固定长度标签和虚电路的技术，在不放弃基于目的地IP数据报转发的基础设施的前提下，当可能时通过选择性地标识数据报并允许路由器基于固定长度的标签（而不是目的地IP地址）转发数据报来增强其功能。

首先考虑由MPLS使能的路由器处理的链路层帧格式。如下图所示：

一个MPLS使能的路由器常被称为标签交换路由器（label-switched router），因为它通过在其转发表中查找MPLS标签，然后立即将数据报传递给适当的输出接口来转发MPLS帧。下图展示了一组MPLS使能路由器之间交互过程：

MPLS使能路由器R1到R4在完成分组的转发时没有接触分组的IP首部。由于篇幅，不讨论MPLS实际如何计算在MPLS使能路由器之间分组的路径，也不讨论它如何收集链路状态信息以用于这些路径计算中。

至此，我们知道MPLS基于标签执行交换，不必考虑分组的IP地址，潜在增加了交换速度，但其优点不在于此。与基于IP地址的分组转发相比，MPLS提供了沿着多条路由转发分组的能力，使用标准IP路由选择协议这些路由将是不可能的。

6.6 数据中心网络

每个数据中心都有自己的数据中心网络，它们将其内部主机彼此互联并与因特网中的数据中心互联。维持一个数据中心的费用庞大，网络创新是减少整体成本和性能最大化的关键。感兴趣可以了解数据中心中的刀片（blade）、机架顶部（Top of Rack，TOR）交换机、边界路由器（border router）等概念。如下图所示：

数据中心网络设计是互联网络和协议设计的艺术，其专注于机架彼此连接和与边界路由器相连。

1.负载均衡

在数据中心内部，外部请求首先被定向到一个负载均衡器（load balancer），其任务是向主机分发请求，以主机当前的负载作为函数来在主机之间均衡负载。由于负载均衡器基于分组的目的端口号和目的IP地址做决策，故常被称为“第四层交换机”。

2.等级体系结构

当主机规模扩展到几万至几十万时，数据中心通常应用路由器和交换机等级结构（hierarchy of router and switch）。如上图30所示。所有链路通常使用以太网作为链路层和物理层协议，并混合使用铜缆和光缆。

另一个趋势是采用船运集装箱的模块化数据中心（Modular Data Center，MDC）。书中做了有限的介绍，此处略。

6.8 小结

本章，我们学习了链路层，包括其服务、支撑它操作的原则和许多重要的特定协议，并实现了链路层服务。

链路层的基本服务是将网络层的数据报从一个节点移动到一个相邻的节点。我们围绕其提供的服务，研究了差错检测和纠正技术、多路访问协议、链路层寻址、虚拟化局域网（VLAN）以及扩展的交换局域网和数据中心网络的构造方法。

后续的章节将讨论无线网络、网络安全和多媒体网络，这些主题跨越了多个层次。让人欣喜的是，目前的学习已经奠定了后续章节的基础。
