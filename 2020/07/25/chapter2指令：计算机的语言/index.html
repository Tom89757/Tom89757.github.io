2.1 引言

计算机语言中的基本单词称为指令，一台计算机的全部指令称为该计算机的指令集。指令集分为两种形式，人们编程书写的形式和计算机识别的形式。本章将自顶向下，从助记符表示的汇编语言过渡到计算机的真实语言；下一章继续向下探究，解释算术运算的硬件以及浮点数的表示方法。

本书选择MIPS指令集，其与ARMv7、Intel x86和ARMv8等指令集具有相似之处，这是因为所有的计算机必须实现一些类似的基本操作。此外，计算机设计者有一个共同目标：找到一种语言，可方便硬件和编译器的设计，且使性能最佳，同时使成本和功耗最低。

MIPS指令集是符合“设备简单性”原则的一种指令集，本章的目标是介绍它如何用硬件表示，以及它与高级编程语言（主要是C，提及Java）的关系。下图展示了MIPS指令集的总体情况：

2.3 计算机硬件的操作数

MIPS算术运算的操作数必须来自寄存器，寄存器由硬件直接构建且数量有限，寄存器大小为32位，也称为一个字。本节中MIPS算术指令的三个操作数必须从32个32位寄存器中选取，其符合硬件设计三条基本原则中的第二条：越小越快。大量的寄存器可能使时钟周期变长，因为电信号传输的距离变长。（此外，指令格式位数同样限制了寄存器个数）

本书中使用$\$s0, \$s1, …$表示变量所对应的寄存器，使用$\$t0, \$t1, …$表示临时寄存器，在2.8节将说明理由。下面展示了上述汇编程序的修正版：

存储器操作数

寄存器个数有限，而复杂数据结构所包含的元素可能非常多，其保存在存储器中，所以MIPS必须包含在存储器和寄存器之间传送数据的指令——数据传送指令。为了访问存储器中的一个字，指令必须给出存储器地址，地址可以看作存储器这个巨型一维数组的下标。

  • 取数（load）指令lw：load word的缩写，如lw $t0, 8($s3) # Temporary reg $t0 gets A[8]。其中8称为偏移量，$s3称为基址寄存器。
  • 存数（store）指令sw：store word的缩写，如sw $t0, 48($s3)。

书中P46对MIPS的细节作了补充：

  1. MIPS按字节编址，故字的起始地址必须为4的倍数；此外，MIPS采用大端编址。
  2. 由于连续字的地址相差4，故上述取数操作中8应改为32。
  3. 将不常使用（或稍后使用）的变量存回寄存器的过程称为寄存器溢出（spilling）。
  4. 寄存器与存储器相比，访问时间短、吞吐率高，寄存器中的数据访问速度快并易于利用，访问寄存器相对于访问存储器功耗更小。

常数或立即数操作数

addi $s3, $3, 4 # $s3 = $s3 + 4。该指令相比于从存储器中取常数，所需指令数变为1，且速度快、能耗低。此外，取数/存数操作同样可视为一个操作数为0的加法，因此MIPS将寄存器$zero恒置为0。

值得补充的是，MIPS支持负常数，所以MIPSbuxuyc设置减立即数的指令。（有趣的是，最初将数组的基地址作为偏移量，而将数组下标存储在寄存器中；随着存储器容量的增大，基地址不适宜作为偏移量，故将其保存在寄存器中）

2.5 计算机中指令的表示

指令在计算机内部是以若干或高或低的电信号的序列表示的，并且形式上和数的表示相同。指令的各部分均可看作一个独立的数，将数拼接在一起就形成指令。前面介绍了寄存器，故计算机需要提供将寄存器映射到数字的方式，在MIPS汇编语言中，寄存器$\$s0 \sim \$s7$映射到寄存器16 ~ 23，寄存器$\$t0 \sim \$t7$映射到8 ~ 15。

故add $t0, $s1, $s2可翻译成纯数字表示的机器指令：0 17 18 8 0 32。此处为十进制表示。对应的二进制表示为：

000000 10001 10010 01000 00000 100000
6位 5位 5位 5位 5位 6位

其中0和32组合起来告诉MIPS计算机该指令完成加法运算，17表示寄存器$\$s1$，18表示寄存器$\$s2$，8表示寄存器$\$t0$，0说明该字段未用到。指令的布局形式称为指令形式，MIPS指令占32位，与数据字的位数相等，符合简单源于规整的原则。

将指令的数字形式称为机器语言，这样的指令序列叫做机器码。

MIPS字段

为了便于讨论，给MIPS字段（R型）命名如下：

  • op：指令的基本操作，通常称为操作码（opcode），指令中用来表示操作和格式的字段。
  • rs：第一个源操作数寄存器。
  • rt：第二个源操作寄存器。
  • rd：用于存放操作结果的目的寄存器。
  • shamt：位移量。2.6节后才会用到。
  • funct：功能。一般称为功能码（function code），用于指明op字段中操作的特定变式。

当某条指令所需比上述字段更长的字段时，问题就会发生，如取字指令中常数的偏移量会限制在$2^5$内，此时需要对上述指令格式进行调整，这就涉及到第三条也就是最后一条硬件设计原则：优秀的设计需要适宜的折中方案。MIPS选择的折中方案是：保持所有的指令长度相同，但不同类型的指令采用不同的指令格式，上述格式称为R型（用于寄存器），下属格式称为I型（用于立即数）：

16位的地址字段意味着取自指令可以取相对基址寄存器地址偏移$\pm 2^{15}$或者说32678个字节（$\pm 2^{13}$或者8192个字）范围内的任意数据字。同样加立即数指令中常数也被限制不超过$\pm 2^{15}$。这也说明了很难设置32个以上的寄存器的原因，因为这意味着rs和rt字段需要额外的位，压缩了地址字段的空间。

2.7 决策指令

计算机与简单计算器的区别在于决策能力。C语言中用if语句或goto语句描述决策，MIPS中类似的语句有：

  • beq register1, register2, L1。其表示若两个寄存器中值相等，则转到标签为L1的语句执行。beq代表branch if equal。
  • bne register1, register2, L1。其表示不相等时跳转。bne代表branch if not equal。

2.8 计算机硬件对过程的支持

过程（procedure）或函数是程序员进行 结构化编程的工具，其有助于提高程序的可理解性和代码的可重用性。过程同样是一种抽象，函数可以看作它的实体。在过程运行过程中，程序遵循以下6个步骤：

  1. 将参数放在过程可以访问的位置。
  2. 将控制转交给过程。
  3. 获得过程所需的存储资源。
  4. 执行需要的任务。
  5. 将结果的值放在调用程序可以访问的位置。
  6. 将控制返回初始点，因为一个过程可能由一个程序中的多个点调用。

寄存器是保存数据最快的位置，所以要尽可能多的使用寄存器，下面是32个寄存器中与过程调用有关的寄存器：

  • $a0 ~ $a3：用于传递参数的4个参数寄存器。
  • $v0 ~ $v1：用于返回值的两个值寄存器。
  • $ra：用于返回起始点的返回地址寄存器。

MIPS汇编语言中还包括跳转和链接指令jal（jump-and-link instruction），其格式为：jal ProcedureAddress：其跳转到某个地址的同时将下一条指令的地址保存在寄存器$ra中。

为了使过程返回到$ra中保存的地址，MIPS中包含寄存器跳转指令jr，用于case语句，表示无条件跳转到寄存器所指定的地址：jr $ra。

通过jal和jr指令实现了调用程序（调用者）跳转到被调用过程（被调用者）再返回到调用程序（调用者）的需求。在存储程序的概念中，使用寄存器保存当前运行的指令地址是必要的，其合理的名字应该为指令地址寄存器，出于历史原因，通常将其称为程序计数器（program counter，PC）。jal指令实际是将PC + 4保存在寄存器$ra中，从而将链接指向下一条指令，为后面返回调用者做好准备。

2.11 并行与指令：同步

任务独立时任务的并行执行容易实现，但当任务间需要协作时，执行读任务的一方需要知道写任务何时完成写操作，才能安全地读数据。这意味着任务间需要同步，否则会发生数据竞争（data race）。

本章重点讨论加锁和解锁同步操作的实现，它们可以创立一个仅允许单个处理器操作的区域，叫做互斥区。由于在CSAPP第12章并发编程中已经描述了该机制，此处不再赘述。（实际上，本书上对同步作了极其晦涩的描述，看不太懂mark）

值得补充的是，在多处理器中实现同步需要一组操作，提供对存储单元进行原子读和原子写的能力，使得在进行存储器原子读或原子写操作时任何其他操作都不得插入。

编译器

编译器将C程序转换成一种机器能理解的符号形式的汇编语言程序（assembly language program）。

汇编器

汇编语言也能够处理一些机器语言指令的常见变种，这类指令称之为伪指令（硬件不需要实现，只是便于理解）。书中还提及了符号表（symbol table），该概念在CSAPP中有极为详细的介绍。

链接器

链接器的出现解决了重复翻译导致的计算资源浪费的问题，它将所有独立汇编的机器语言程序“拼接”在一起。同样，CSAPP有详细介绍，此处不再赘述。

加载器

在CSAPP中并未作单独介绍。它用于将磁盘中的可执行文件读入内存并启动执行。在UNIX系统中，加载器（loader）按照如下步骤工作：

  1. 读取可执行文件头来确定代码段和数据段的大小。
  2. 为正文和数据创建一个足够大的地址空间。
  3. 将可执行文件中的指令和数据复制到内存中。
  4. 把主程序的参数（如果存在）复制到栈顶。
  5. 初始化机器寄存器，将栈指针指向第一个空位置。
  6. 跳转到启动例程，它将参数复制到参数寄存器并且调用程序的main函数。当main函数返回时，启动例程通过系统调用exit终止程序。

x86 总结

Intel的16位微处理器比它的竞争对手的同层次体系结构（如 Motorola 68000）早两年问世，这个领先使得IBM选用8086作为其PC的CPU。虽然x86体系结构的制造比MIPS和ARM更为困难，但广大的市场推动了它的迅速开发。尽管x86在PC中占据绝对优势，但在个人移动设备里面还不具有竞争力。

与x86中使寄存器加宽到64位只做了很小的改变相比，ARM中作了完全的改进。甚至于相较于ARMv7，ARMv8与MIPS更为相似。ARMv8的具体细节不是目前学习的重点，此处略mark。

2.19 谬误与陷阱

  • 谬误1：更强大的指令意味着更高的性能

在x86指令集中，使用复杂指令的复制操作虽然所需指令数只需一条，但使用两条指令集标准指令（先取到寄存器再存回寄存器）要比其快2倍。

  • 谬误2：使用汇编语言编程来获得最高的性能

事实上，经过对编译器的不断改进，编译器产生的代码与手工编写的代码再性能上的差距正在快速缩小。不从事编译器领域工作的程序员无需精通汇编程序的编写，因为编译器能比程序员更好地分配寄存器。（手工编写汇编代码的劣势还在于：需要更多时间编码和调试，可移植性差，难以维护，并且需要随着硬件和操作系统的迭代不断更新）

有趣的是，软件工程中少数几个倍广泛接收的公理之一便是：编写的程序行数越多所花的时间也越多。这也与汇编编程的式微不谋而合。

  • 谬误3：商用集计算机二进制兼容的重要性意味着成功的指令集不需要改变

2.20 本章小结

存储程序计算机的两个准则是：指令的使用与数字没有区别；使用可修改的存储器。优秀的指令集需要精妙地平衡程序执行所需的指令数目、指令执行所需的时钟周期数和时钟速度。有3条硬件设计的原则可以知道设计者：

  1. 简单源于规整。
  2. 越小越快。
  3. 优秀的设计需要好的折中。

在指令集设计中同样蕴含了“加大大概率事件”的伟大思想，如条件分支的PC相对寻址和大常数操作数的立即数寻址。下图展示了到目前为止我们讲述的MIPS指令，包括实际指令和伪指令：

每一类MIPS指令与编程语言中出现的结构有关：

  1. 算术指令对应于赋值语句中的运算。
  2. 传输指令很可能发生在处理像数组和结构体这样的数据结构时。
  3. 条件分支被用于if语句和循环。
  4. 无条件分支被用于过程调用和返回以及case/switch语句。
