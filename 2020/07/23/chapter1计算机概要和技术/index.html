1.1 引言

随着计算机科学的发展，现在已经出现了科学探索的新方式，即计算机科学家联合理论和实验科学家，共同探索天文学、生物学、化学和物理学的前沿问题。

随着计算机革命进行，车载计算机、收集、人类基因项目、万维网、搜索引擎等过去幻想的事物映射到了现实。

计算应用的分类及其特性

不同的计算应用使用通用的计算技术，但服从不同的设计需求。

  • 个人计算机：对普通用户来说有足够用的性能，价格也能够负担，运行第三方软件。

  • 服务器：适用于大负载任务，一般通过网络访问，由多个用户并行使用。制造技术同个人计算机类似，在计算、存储和I/O能力上进行升级，强调可靠性。（超级计算机同样属于服务器，但市场份额很小）

  • 嵌入式服务器：数量最多的计算机，应用和性能范围广泛，包括汽车、电视中的微处理器以及用来控制飞机和货船的处理器网络。其设计目标为运行单一应用程序或者一组相关的应用程序，通常和硬件集成在一起以单一系统的方式一并交付用户。此外，它对成本、功耗以及故障等方面有严格要求。

欢迎来到后PC时代

  • 个人移动设备（Personal mobile device, PMD）部分替代了传统PC，当前为手机或平板，以后可能还有电子眼镜。PMD是连接到网络上的小型无线设备，由电池供电。
  • 云计算替代了传统服务器，其依赖于称之为仓储规模计算机（Ware-house Scale computer, WSC）的巨型数据中心，中小型公司可以租用WSC为PMD提供软件服务。

PMD和WSC是硬件工业的革命，而通过云计算实现的软件即服务（Software as a Sevice, SaaS）（在网络上以服务的方式提供软件和数据，通过本地客户端登录到远程服务器上执行）是软件工业的革命。

你能从本书中学到什么

现代计算机的特征是处理器的并行性和内存的层次性，内存容量已不像上世纪深受限制，而需要考虑的是运行在PMD或云上的程序的能效，这意味着程序员需要挖掘藏在代码之下的细节，即提高对计算机组成的认知。本书旨在解决如下问题：

  • 程序性能的本质：高级语言如何翻译为硬件之间的语言？硬件如何执行程序？
  • 硬软件接口：它是硬件和软件的桥梁，软件如何通过它指导硬件完成功能？
  • 性能决定因素：程序性能取决于原始程序，将该程序转换为计算机语言的软件（编译器、汇编器、链接器等）以及执行该程序的硬件的有效性。（但这不在考试的范围之内）
  • 改进性能：硬件设计者用于改进性能的技术？本书将介绍现代计算机设计的基本概念，深入了解可阅读《Computer Architecture: A Quantitative Approach》（这也不在考试范围内）
  • 改善能效：硬件设计者和程序员分别可以使用什么技术？
  • 并行处理：为什么串行处理发展为并行处理？其结果是什么？
  • 伟大思想：计算机1951年面世以来计算机架构师提出了哪些伟大思想构成现代计算机基础？

回答了这些问题，提升程序性能或评估问题解决方法将不再是反复实验的不确定过程，而是深入原理的科学过程。

本书将对术语给出明确定义（某些术语通常以首字母缩略词出现）。由于计算机的最终目标为提供程序性能，故对一个C程序（一个矩阵和一个向量相乘）的性能优化过程将贯穿始终，最终其性能能够提升200倍。

1.2 计算机系统结构中的8个伟大思想

这些思想历久弥新，在本书中将明确使用近100次。

面向摩尔定律的设计

单芯片上的集成度每18~24个月翻一番。计算机设计需要几年时间，而计算机设计者必须预测其设计完成时的工艺水平，故摩尔定律的背后本质是设计者所定芯片集成度目标的提升。

使用抽象简化设计

抽象是用于提高硬件和软件生产率的主要技术之一，抽象用来表示不同的设计层次，在高层次中看不到低层次中的细节，只能看到简化模型。

加大大概率事件

加速大概率事件远比优化小概率事件更能提高性能。这意味着设计者需知什么事件经常发生，其需要实验和评估确定。该思想与Amdahl定律类似，后者旨在指出：若想显著提高性能，需要对程序的大部分进行优化。

通过并行提高性能

无论是单核上进程的并发，还是多核上线程的并行，都是并行提升性能的方式。

通过流水线提高性能

流水线是一个特别的并行性场景。

通过预测提高性能

当预测准确率较高且预测错误处罚不大时，采用该策略可以有效减少时钟周期数。

存储器层次

当今存储器速度限制了CPU性能的发挥，通过存储器层次可以解决CPU对主存速度的需求。程序员看到的主存同时具有存储器层次中顶层的高速度和低层中的大容量和廉价。

通过冗余提高可靠性

由于每个物理器件都可能失效，因此通过使用冗余不见的方式提高系统的可靠性，冗余不见可以替代失效部件并帮助检测错误。

从复杂的应用程序到简单的指令需要经过几个软件层来讲复杂的高层次操作逐步解释或翻译成简单的计算机指令，这同样是抽象的体现。下图展示了硬件和软件的层次结构：

操作系统和编译程序对系统软件是必须的，操作系统是用户程序和硬件之间的接口，其最为重要的作用是：

  • 处理基本的输入和输出操作。
  • 分配外存和内存。
  • 为多个应用提供共享计算机资源的服务。

从高级语言到硬件语言

计算机语言即二进制数，计算机指令则是能被计算机识别并执行的位串。从第一代程序到如今，程序员与计算机交流的语言从二进制的机shy器语言，到以助记符形式表示的汇编语言，再到抽象层次更高的高级编程语言（同样体现了抽象）。（机器语言和汇编语言均为机器指令，只是表示形式不同）下图展示了三种语言形式的实例：

使用高级编程语言有三种好处：

  1. 使程序员以更自然的语言来思考，所写的代码更易于理解，并且可按用途来进行设计。
  2. 提高了程序员的生产率，即用短短几行即可表示出设计用意。简明性是其相对汇编语言最大的优势。
  3. 提高了程序相对于计算机的独立性，编译器和汇编程序能够将高级语言程序翻译成任何计算机的二进制元指令。

1.4 硬件概念入门

任何一台计算机的基础硬件都要完成相同的基本功能：输入数据、输出数据、处理数据和存储数据。本书的目的即描述这些功能如何完成。组成计算机的5个经典部件是输入、输出、存储器、数据通路（运算器）和控制器，其中最后两个部件合称为处理器，我们总能将任何计算机中的任何部件归于这五种之一。下图展示了计算机的硬件组成：

显示器

大多数个人移动设备都用液晶显示（LCD）来获得轻巧、低功耗的显示效果。LCD大多使用动态矩阵显示技术，其每个像素都由一个晶体管精确地控制电流，使图像更清晰。书上关于显示器原理有更具体的介绍，此处略。

触摸屏

后PC时代的平板电脑和手机使用触摸屏替代了键盘和鼠标，用户界面更为友好。触摸屏有多种实现方法，其中一种为电容感应（利用了人为导体的特性）。

计算机中内存由DRAM芯片组成，用来承载程序的指令和数据，DRAM访问任何位置的数据所需事件基本相同（串行访问磁带则不是）；处理器内部则使用缓存，其更小更快，由SRAM组成。

计算机中最重要的抽象之一是硬件和底层软件之间的接口，其被命名为指令集体系结构，提供给程序员的基本指令集和操作系统接口合称为应用二进制接口。计算机体系结构使得计算机设计者可以不必考虑硬件即可讨论功能（硬件的实现必须按照体系结构的抽象），指令集体系结构使得同一软件可以由成本不同、性能也不同的实现方法来完成。从中我们可以总结出这样一种伟大思想：无论硬件还是软件都可以抽象分为多个层次，每个较低的层次把细节对上层隐藏起来。

1.6 性能

硬件和软件的复杂和先进使得性能评价比较困难，同时精确地测量和比较不同计算机之间的性能对购买者和设计者都很重要，这意味着评价计算机性能的挑战性和必要性。

性能的定义

书中通过对客机的性能展示，表明对不同的性能衡量因素各客机表现各异，对计算机来说也同样如此。衡量计算机性能的两个标准分别为响应时间和吞吐率。实际上，响应时间和吞吐率往往互相影响，尤其是响应时间的减少几乎都意味着吞吐率的增加，因此本书前几章主要考虑响应时间方面。

性能的度量

书上进一步对响应时间（更易于理解的叫法是墙上时钟时间和消逝时间）和CPU执行时间作了定义和区分，后者进一步可分为系统CPU时间和用户CPU时间，后者只表示在CPU上花费的时间但用户难以感知，前者则相反。

书中接着对响应时间和用户CPU时间所衡量的性能作了区分，系统性能表示空载系统的响应时间，CPU性能表示用户CPU时间。本章将重点放在CPU性能上。（需要明确的是，不同的应用关心计算机性能的不同方面）

为更准确的衡量计算机硬件性能，我们使用时钟周期（单位通常为ps）、时钟频率（单位通常为GHz）等概念。

CPU性能及其因素

两个简单但有用的公式（二者等价）：

一个程序的CPU执行时间 = 一个程序的CPU时钟周期数 x 时钟周期时间；

一个程序的CPU执行时间 = 一个程序的CPU时钟周期数 / 时钟频率。

指令的性能

由于程序的运行最后为指令的执行，故CPU执行时间依赖于程序中的指令数，下面是时钟周期数与指令数的关系：

CPU时钟周期数 = 程序的指令数 x 每条指令的平均时钟周期数

术语CPI（clock cycle per instruction）表示执行每条指令所需的时钟周期数的平均值，CPI是一个程序全部指令所用时钟周期数的平均值。

经典的CPU性能公式

由上述公式可推出：

CPU时间 = 指令数 x CPI x 时钟周期时间

CPU时间 = 指令数 x CPI / 时钟频率

从而建立其性能与指令数、CPI和时钟周期三个因素间的关联。但是唯一可靠衡量性能的指标只有时间，下图展示了基本的性能指标及其测量单位：

1.7 功耗墙

下图展示了30年间Intel八代微处理器的时钟频率和功耗的增长趋势：

可以看出功耗已经到达了极限，再高的话无法让处理器冷却下来（需要补充的是，使用能耗比使用功耗来评价功耗更加合理）。目前占统治地位的集成电路技术是CMOS（互补型金属氧化半导体），其主要能耗来源是晶体管的状态从0翻转到1或从1翻转到0消耗的能量。能耗和功耗满足以下公式：

能耗 $\propto$ 1/2 x 负载电容 x ${电压}^2$

功耗 $\propto$ 1/2 x 负载电容 x ${电压}^2$ x 开关频率

其中负载电容与晶体管数量有关，开关频率与时钟频率有关。上图可以看出，虽然时钟频率增加了1000倍，功耗只增加了30倍，这是因为每次工艺更新换代时都会降低电压，20多年来电压由5V降到了1V（但电压继续下降会使晶体管泄漏电流过大，目前40%的功耗是由泄漏造成的）。似乎物理学（无法继续降低电压）和成本（功耗过高冷却成本大幅提升）的限制使得功耗的增加到达了瓶颈，形象地将之称之为功耗墙。

1.8 沧海巨变：从单处理器向多处理器转变

功耗极限迫使微处理器的涉及产生巨变，工业界开始在单片微处理器中加入多个处理器，以求更大的吞吐率，不再追求降低单个程序运行在单个处理器上的响应时间。一个“四核”微处理器（microprocessor）是一个包含了4个processor或者说4个core的芯片。

多核的架构意味着程序员需要重写它们的代码来发挥多核的优势，而不是像以往只依赖硬件、体系结构和编译程序的创新（无需修改一行代码）即可享受程序性能按照摩尔定律暴涨。正因为此，我们更需要了解软件和硬件系统的协同工作，即本书的核心——硬件/软件接口。

硬件上的并行迫使程序员需要显式地按并行方式重写其程序，但其推广有以下困难：

  1. 并行编程提高了编程难度，在确保解决问题的同时还需要提高程序运行速度，并且需要为用户提供可稳定使用的统一接口。
  2. 为发挥硬件性能，需要将应用划分为大致均等的任务，并减少调度开销。（一个好的类比是让8个记者完成一篇报道，其开销包括：调度、负载平衡、通信以及同步）

后面的4章中每章至少有一节都将呈现有关并行性革命的内容，第6章将用一整章介绍并行编程。

1.10 谬误和陷阱

本书每一章都会有“谬误与陷阱”一节，其目的是说明我们在实际中经常遇到的误解和容易犯的错误，即谬误和陷阱。

  • 陷阱1：在改进计算机的某个方面时期望总性能的提高与改进大小成正比。

由该陷阱可提出Amdahl定律，该定律阐述了“特定改进的性能提升可能被进行的改进占程序的比例所限制”的规则，它是“收益递减定律”的量化版本。如一个执行时间100s的程序中80s对应指令的优化的极限是将该程序性能提高5倍。Amdahl定律和CPU性能公式结合，是一种很方便的性能评价工具。

  • 谬误1：计算机利用率低时功耗也低。

事实上，在最好的测试结果中，10%的工作负载即可消耗1/3的最大功耗，故有科学家提出需要对硬件重新设计，使得工作负载与功耗成线性关系，即10%的工作负载消耗10%的峰值功耗。

  • 谬误2：面向性能的设计和面向能量效率的设计具有不相关的目标。

对软硬件的优化而言，即使优化后能耗升高，但运行时间减少，整体上可能仍然节约了能量。

  • 陷阱2：用性能公式的一个子集取度量性能。

简单使用时钟频率、指令数和CPI中的一种或两种因素比较性能是不科学的，其本质是没有用唯一的性能指标——执行时间来衡量性能。有一种用MIPS取代时间度量性能的方法，对给定程序，MIPS表示为：MIPS = 指令数 / （执行时间 x $10^6$）。它规定了性能与执行时间成反比，但是存在三个问题（问题的本质在于使用MIPS并未考虑到不同计算机的指令集体系结构的不同）：

  1. 规定了指令执行的速率，但没有考虑同一程序在不同计算机中指令数的差异。故无法比较不同指令集的计算机。
  2. 在同一计算机上，不同的程序会有不同的MIPS，因而一台计算机不会只有一个MIPS值。（个人认为是废话，因为CPU时间同样随任务不同变化）
  3. 如果一个新程序的指令数更多，但每条指令的执行速度更快，则MIPS的变化与性能无关。

1.11 本章小结

硬件和软件设计者都是用分层的方法构建计算机系统,每个下层都对上层隐藏本层的细节，即使用抽象，最重要的抽象层次是硬件和底层软件之间的接口——指令集体系结构。

同样，书中描述了唯一用来衡量性能的计算CPU时间的公式，因为执行时间是唯一有效且不可推翻的性能度量方法。为提高性能，发展出了两种重要新思想：其一是并行编程；其二是利用存储器层次结构中的访问局部性。这两种思想在硬件和软件层面交织，是提高性能的必要途径。

在抽象的底部是计算机的5个经典部件，它们也是本书后面几章的框架：

  • 数据通路：第3、4、6章和附录C
  • 控制器：第4、6章和附录C
  • 存储器：第5章
  • 输入：第5章和第6章
  • 输出：第5章和第6章
