chapter1引论

发表于 2020-07-11 分类于 读书笔记

本章将阐述学习本书的目的，简要回顾离散数学和程序设计的概念，具体为：

  • 程序在较大输入时的运行性能和适当输入时的运行性能同等重要。
  • 总结学习本书所需要的数学基础。
  • 简要复习递归。

1.1 本书讨论的内容

书中抛出了一个选择问题“在N个数中，选出其中的第k大的数”，并给出了两种思路：第一种思路是使用冒泡排序对所有数进行排列；第二种思路是先将前k个数组成有序数组，再将剩下的N-k个数与数组中的每个数比较，并归位。这两种思路在规模较大时均不合格，书中第7章给出了更高效的解法。

抛出的另一个问题是“寻找二维字母表中的单词”，同样有两种思路：第一种思路是检查每个有序三元组（行，列，方向）；第二种思路是检查有序四元组（行，列，方向，字符数）。同样在规模较大时这两种思路难以令人接受，后续会给出合理的算法。

本书将看到：在未具体编码时对于大量输入如何估计程序的运行时间；找到彻底改进程序速度以及确定程序瓶颈的方法。

1.3 递归简论

递归运算往往意味着先处理基准情形；对于数值运算使用递归是不被期望的；递归调用和其他的调用没有本质不同，无非是很多函数调用被挂起，等待与之对应的函数调用的返回值；当递归调用无法递归到基准情形时，就会导致无穷调用，直到程序崩溃。以上递归性质可以推导出递归的两个基本法则：

  • 基准情形：递归必须有基准情形，它们不需要进行递归调用即可求解。
  • 不断推进：递归调用必须向产生基准情形的方向不断推进。
  • 设计法则（由下面实例得出）：假设所有的递归调用都能运行。这意味着不需要执着于递归

chapter12并发编程

发表于 2020-07-09 更新于 2020-07-11 分类于 读书笔记

如第8章所说，如果逻辑控制流在时间上重叠，它们就是并发（concurrency）的。并发出现在计算机系统的不同层次，如Linux信号处理程序、硬件异常处理程序和进程。并发并不限于内核，应用级程序中也大有可为。

  • 访问慢速I/O设备：等待I/O设备（如磁盘）数据到达时，可以运行其他进程，使CPU保持繁忙。
  • 与人交互：如在使用大型带UI的应用程序时进行的多种操作。
  • 通过推迟工作降低延迟：如对动态内存分配器来说可以推迟合并，降低单个free操作的延迟。
  • 服务多个网络客户端：创建并发服务器，为每个客户端创建单独的逻辑流。
  • 在多核机器上进行并行计算：使多个流并行执行，而不是交错执行。
  • 进程：每个逻辑控制流都是一个进程，由内核来调度和维护。
  • I/O多路复用：程序是一个单独的进程，程序中所有的复用流共享同一个地址空间。
  • 线程：线程是运行在一个单一进程上下文中的逻辑流，由内核进行调度，同时像I/O复用流一样共享一个虚拟地址空间。

本章研究上述三种不同的并发编程技术——进程、I/O多路复用和线程。以一个并发网络服务器作为贯穿全章的应用程序。

chapter9虚拟内存

发表于 2020-07-05 更新于 2020-07-11 分类于 读书笔记

一个系统中的进程与其他进程共享CPU和主存资源。为了有效地管理内存并且少出错，现代系统提供了一种对主存的抽象概念——虚拟内存（VM）。它是硬件异常、硬件地址翻译、主存、磁盘文件和内核软件的完美交互，他为每个进程提供了一个大的、一致的和私有的地址空间。

虚拟内存提供了三种重要的能力：

  • 把主存看成是一个存储在磁盘上的地址空间的高速缓存，主存中只保留活动区域在磁盘和主存间传送数据。
  • 为每个进程提供了一致的地址空间，简化内存管理。
  • 保护每个进程的地址空间不被其他进程破环。

程序员需要理解它处于以下原因：

  • 它是核心。虚拟内存遍布计算机系统的所有层面。
  • 它很强大。通过它可以销毁内存片、将内存片映射到磁盘文件的某个部分，和其他进程共享内存。
  • 它很危险。程序分配地址空间时都会与它交互，使用不但会导致严重错误。

本章讲述虚拟内存如何工作，以及如何在程序中使用和管理虚拟内存和与内存相关的错误。

chapter8异常控制流

发表于 2020-07-04 更新于 2020-07-11 分类于 读书笔记

从处理器通电开始，到断电为止，程序计数器PC中的值不断地在指向不同指令的地址间转换，可以将这些地址的值按顺序组成一个序列，这种转换称为控制转移，这个序列称为处理器的控制流。

与内部程序无关的系统状态的变化需要控制流发生突变来进行回应，内部程序中的跳转、调用和返回指令同样回导致突变。这些突变被称之为异常控制流（Exceptional Control Flow, ECF），它可以发生在计算机系统的各个层次，硬件层、操作系统层、应用层。

对程序员来说，理解ECF的好处有：

  • 帮助理解重要的系统概念。ECF是操作系统用来实现I/O、进程和虚拟内存的基本机制。
  • 帮助理解应用程序如何与操作系统交互。
  • 帮助编写有趣的新应用程序。如Unix shell和Web服务器。
  • 帮助理解并发。理解ECFui理解并发的第一步。
  • 帮助理解软件异常如何工作。软件异常允许程序进行非本地跳转来响应错误情况，非本地跳转是一种应用层ECF。

目前已经了解应用如何与硬件交互，本章将学习应用如何与操作系统交互，这些交互都围绕ECF展开。异常位于硬件与操作系统交界处，而进程和信号位于应用和操作系统交界处。

chapter7链接

发表于 2020-07-02 更新于 2020-07-11 分类于 读书笔记

链接是将各种代码和数据片段收集并组合成为一个单一文件的过程，这个文件可以被加载到内存并执行。链接过程可以执行在编译、加载甚至运行时，它使得分离编译得以实现。理解链接有以下好处：

  • 帮助构造大性程序。当构造大性程序时经常会遇到缺少模块、缺少库或者不兼容的库版本等引起的链接器错误。
  • 帮助避免一些危险的编程错误。如错误的定义多个全局变量时。
  • 帮助理解语言的作用域规则如何实现。全局和局部对程序来说到底意味着什么。
  • 帮助理解其他的系统概念。如加载和运行程序、虚拟内存、分页、内存映射。
  • 可以利用共享库。许多软件产品在运行时通过共享库升级程序；Web服务器依赖于共享库的动态链接提供动态内容。

chapter6存储器层次结构

发表于 2020-07-01 更新于 2020-07-11 分类于 读书笔记

目前来看，我们对计算机的认识依赖于一个简单的模型，CPU执行指令，存储器血死为CPU存放指令和数据，并且将存储器看作一个线性的字节数组，CPU可以在常数时间内访问每个存储器为止。

实际上，存储器系统时一个具有不同容量、成本和访问时间的存储设备的层次结构，而CPU寄存器保存着最常用的数据。正如第一章所说，小的、速度更快、高一级的存储器可以看作大的、速度较慢、低一级的存储器的缓冲区域（此处可以对存储器速度作实际的说明，指令执行过程中，对寄存器访问周期数为0，高速缓存为4~75，主存上百，磁盘千万数量级）。

要使得程序运行具有更快的速度，则需要将其数据项存储在层次结构中较高的地方，以使CPU更快的访问，这体现了计算机程序的局部性（locality）属性。

本章将讲述基本的存储技术——SRAM、DRAM、ROM以及旋转的和固态的硬盘，重点是高速缓存存储器，它们是链接CPU和主存的桥梁。
