chapter7排序

本章讨论数组的排序问题，为了简化问题，数组中的元素均为整数。本章主要讨论在主存中完成的内部排序，这意味着排序元素较少（$\leq 10^6$），在本章末尾将讨论在磁盘上完成的外部排序。

对内部排序的讨论将包含：

  1. 存在一种容易的算法以$O(N^2)$排序，如插入排序。
  2. 一种在实践中有效的算法——希尔排序，以$o(N^2)$运行。
  3. 稍微复杂的以$O(N \log N)$运行的算法。
  4. 任何通用的排序算法均需要$\Omega(N \log N)$次比较。

排序算法收到青睐的原因在于，它们中可能包含一些有趣的、重要的代码优化和设计思想，且对排序算法可以作精确的分析。

chapter5散列

第4章讨论了查找树ADT，它支持对一组元素进行各种操作。而本章讨论的散列表ADT只支持二叉查找树允许的部分操作。

散列表的实现叫做散列，散列是一种以常数平均时间执行插入、删除和查找的技术，但是不支持需要元素间排序信息的操作。具体说，第４章中的FindMin、FindMax以及以线性时间将排过序的整个表进行打印的操作都是散列表所不支持的。

本章我们将：

  1. 描述实现散列表的几种方法。
  2. 分析比较这些方法。
  3. 介绍散列表的多种应用。
  4. 将散列表和二叉查找树进行比较。

chapter12并发编程

如第8章所说，如果逻辑控制流在时间上重叠，它们就是并发（concurrency）的。并发出现在计算机系统的不同层次，如Linux信号处理程序、硬件异常处理程序和进程。并发并不限于内核，应用级程序中也大有可为。

  • 访问慢速I/O设备：等待I/O设备（如磁盘）数据到达时，可以运行其他进程，使CPU保持繁忙。
  • 与人交互：如在使用大型带UI的应用程序时进行的多种操作。
  • 通过推迟工作降低延迟：如对动态内存分配器来说可以推迟合并，降低单个free操作的延迟。
  • 服务多个网络客户端：创建并发服务器，为每个客户端创建单独的逻辑流。
  • 在多核机器上进行并行计算：使多个流并行执行，而不是交错执行。
  • 进程：每个逻辑控制流都是一个进程，由内核来调度和维护。
  • I/O多路复用：程序是一个单独的进程，程序中所有的复用流共享同一个地址空间。
  • 线程：线程是运行在一个单一进程上下文中的逻辑流，由内核进行调度，同时像I/O复用流一样共享一个虚拟地址空间。

本章研究上述三种不同的并发编程技术——进程、I/O多路复用和线程。以一个并发网络服务器作为贯穿全章的应用程序。
