数据库由相互关联的数据集合和用于访问它们的程序组成。设计数据库其目的是管理大量信息，信息的管理又涉及三个方面：

  1. 信息存储结构的定义
  2. 信息操作机制的提供
  3. 信息的安全性保证

1.1 数据库系统的应用

企业、银行和大学。

1.2 数据库系统的目标

在数据库系统(DBS)出现之前，普遍使用操作系统的文件处理系统来存储和更新信息。其弊端在于：

  1. 数据的冗余和不一致(data redundancy and inconsistency)
  2. 数据访问困难(difficulty in accessing data)
  3. 数据孤立(data isolation)
  4. 完整性问题(integrity problem)
  5. 原子性问题(atomicity problem)
  6. 并发访问异常(concurrent-access anomaly)
  7. 安全性问题(security problem)

本书中对问题的讨论大多基于在DBS在大学信息管理中的应用。

1.3 数据视图

在用户看来，系统隐藏了关于数据存储和维护的细节，即只给用户提供了抽象出来的管理数据的接口

1.3.1 数据抽象

开发人员通过以下层次的抽象将用户屏蔽在DBS本身的复杂性之外：

  1. 物理层(physical level)：最底层的开发人员会接触这一层
  2. 逻辑层(logical level)：程序设计人员和数据库管理员会接触到这一层。
  3. 视图层(view level)：普通用户会接触这一层，实际上其看到的只是数据库的某个部分。

1.3.2 实例和模式

某时刻存储在数据库中的信息集合称为一个实例(case)，数据库的总体设计称为模式(schema)。其关系有点类似于进程和程序、变量声明和某一时刻变量值。与上述数据抽象层次对应，模式分为：

  1. 物理模式(physical schema)
  2. 逻辑模式(logical schema)
  3. 子模式(subschema)

应用程序的开发大多基于逻辑模式进行。若应用程序不依赖物理模式进行开发，则称其具有物理数据独立性(physical data independence)

1.3.3 数据模型

数据模型(data model)是数据库结构的基础，是一个描述数据、数据联系、数据语义以及一致性约束的概念工具的集合，提供了一种描述物理层(?)、逻辑层和视图层数据库设计的方式。可分为四类：

  1. 关系模型(relational model)：用表的集合表示数据之间的关系，使用最为广泛，2~8章详细介绍
  2. 实体-关系模型(entity-relationship model)：即E-R模型。第7章详细探讨
  3. 基于对象的数据模型(object-based data model)：面向对象的程序设计推动其发展，可看作E-R模型增加了封装、方法(函数)和对象标识等概念后的扩展
  4. 半结构化数据模型(semistructured data model)：允许相同类型的数据项含有不同的属性集。可扩展标记语言(eXtensible Markup Language, XML)广泛用于表示半结构化数据。

1.4 数据库语言

DBS提供数据定义语言(data-definition language, DDL)和数据操纵语言(data-manipulation language, DML)。二者共同构成单一的数据库语言(如SQL)。

1.4.1 数据操纵语言(DML)

它使得用户可以对使用特定数据模型组织的数据进行检索、插入、删除和修改。分为两种：

  1. 过程化DML(procedural DML)：用户需指定需要什么数据和如何获得这些数据
  2. 声明式DML(declarative DML, 也称为费过程化DML)：用户只需指定需要什么数据。故DBS需要自行选择一种访问数据的高效途径。

为了更快的操纵数据，我们必须在物理层定义高效访问数据的算法。

1.4.2 数据定义语言(DDL)

DBS的存储结构和数据访问方式通过一系列DDL语言来说明，其对用户不可见。存储在DB中的数据值需要满足一致性约束(consistency constraint)，DDL提供了指定这种约束的工具。当DB被更新时，DBS需要检查这些约束：

  1. 域约束(domain constraint)：声明某种属性属于某种具体的域相当于约束它可以取的值，如整型、字符型、日期/时间型。
  2. 参照完整性(referential integrity)：一个关系中给定属性集上的取值也在零一关系的某一属性集的去之中出现，如course所在的dept_name必须出现在某一department的dept_name。
  3. 断言(assertion)：一个断言就是DB需要时刻满足的约束，域约束和参照完整性约束是断言的特殊形式。如：每一学期每一个系至少需要开设5门课程。
  4. 授权(authorization)：对用户加以区别，区别当然体现在他们能访问和访问到的数据。如：读权限(read authorization)、插入权限(insert authorization)、更新权限(update)和删除权限(delete)，与DML语言的四种功能对应。

DDL以一些语句作为输入，产生一些输出，输出存放在数据字典(data dictionary)中，数据字典中包含了元数据(metadata，数据的数据)，常规用户不能访问。在读取和修改实际的数据前，DBS先要参考数据字典。
